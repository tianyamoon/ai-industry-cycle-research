## 目录分工

### 01_当前接管入口

只保留当前最值得先读的少量交接材料。

默认原则：

1. 不超过 1-3 份
2. 必须能代表当前主线或当前最活跃专题
3. 必须能让新会话快速接上

### 90_历史归档

历史交接材料统一放这里。

它们保留研究轨迹，但不再和当前入口争位置。

---

## 什么时候才需要新增交接包

只有在以下情况，才建议新增：

1. 主线顺序发生变化
2. 某个专题形成阶段收口
3. 某组样本完成后，下一步接管路径明显改变
4. 如果不写交接包，下一会话很容易误判当前状态

如果只是补一条证据、补一个字段、补一张卡，优先回写原文件，不要默认新建交接包。

---

## 以后交接包应写多厚

默认写成轻量版，只保留四段：

1. 本次推进
2. 关键文件
3. 未决问题
4. 下一步建议

需要补证据时，把证据放回过程层、桥接层或结果层原文，不要把 `99_交接包` 再写成第二套正文。
