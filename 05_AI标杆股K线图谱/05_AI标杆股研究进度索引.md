## 四、当前跨组判断

### 事实

1. 第一批四组样本和第二批 HBM_高性能存储代理组、先进封装组都已出现真实收入、利润或经营验证，不再停留在纯主题导入期；
2. 样本角色已经扩展到核心资产、弹性资产、上游材料、基础设施平台、国产算力、应用平台，以及平台型接口资产、高弹性存储 / 封测资产、平台型先进封装龙头和高弹性先进封装协同资产；
3. 现金流、毛利率、CapEx、生态、付费、客户结构、库存、采购强度和在建工程开始成为比收入增速更重要的验证变量；
4. `2025Q4 -> 2026Q1`，10 个第一批样本公募重仓全部转为减仓，但光模块龙头、沪电和寒武纪的持仓占流通股比仍显著偏高。

### 判断

当前最适合写进上层研究结论的话是：

> AI 第一批样本当前已经从“验证主题是否真实”进入“分层审计利润、毛利率、现金流、CapEx、生态和付费”的阶段。

更细一点看：

1. 光模块组更像主升扩散后段的瓶颈利润链；
2. PCB_CCL组更像高端板兑现后向上游材料传导；
3. 服务器_温控组更像主链核心制造高位验证与液冷基础设施兑现错位并行；
4. 国产算力_应用组更像“供给端核心资产先跑，应用端再看真实付费”的错位验证；
5. HBM_高性能存储代理组更像“平台型互连接口先高位验证，高弹性存储 / 封测资产后续更强但也更危险”的分层兑现；
6. 先进封装组更像“平台型龙头先高位验证，高弹性协同资产利润更强但也更快进入 CapEx 审计”的分层兑现；
7. 资金层最适合的写法已经不是“继续单边加仓”，而是“高位拥挤仍在，但资金开始兑现与再平衡”。

### 待验证点

1. 还不能把四组统一写成“已经进入价格战”；
2. 还不能把高壁垒直接写成“可穿越”；
3. 还不能把当前高利润或高活跃直接外推为长期高回报；
4. 还不能把 HBM_高性能存储代理组直接写成“纯 HBM 龙头映射”；
5. 还不能把先进封装组直接写成“天然可穿越平台”；
6. 还不能把十卡判断和第二批前两组判断直接当成整个 AI 产业链终局结论；
7. 还不能把北向旧接口残值写成最新北向验证结论。

## 五、下一批建议

### 事实

当前第一批十张单卡、总判断、估值与资金验证总表、总映射正式版已经形成，第二批 HBM_高性能存储代理组与先进封装组也已经完成，下一步最自然的推进方向是：

1. 算力设备组双卡执行；
2. 后续失败样本 / 阴影样本组执行；
3. 北向关键样本核验与持续跟踪。

### 判断

下一步应进入：

`HBM_高性能存储代理组 -> 先进封装组 -> 算力设备组 -> 失败样本 / 阴影样本组 -> 关键资金缺口补证`

原因：

1. 四组已经覆盖了硬件、基础设施、国产算力和应用平台；
2. 十卡总判断已经把第一批样本压成了可复用阶段判断；
3. HBM_高性能存储代理组已经补上“高性能存储利润外溢”机制，先进封装组已经补上“高壁垒平台与高弹性协同资产的 CapEx 审计”机制；
4. 当前最自然的动作是继续按固定顺序补算力设备与失败样本；
5. 估值、公募与成交结构已补到可直接引用，且已经上收为总映射正式版，北向缺口应继续并行核验。

### 待验证点

1. 国产算力的真实出货、生态成熟度和费用结构仍需持续验证；
2. AI 应用的真实付费、ARPU 和续费仍不能只靠单季数据判断；
3. 北向最新日度个股统计仍待恢复或换源核验；
4. 第二批后续三组做完后，再决定是否继续扩大 AI 样本宇宙。

## 六、第二批规划入口

### 事实

`AI第二批样本扩展规划_第一版.md` 已经把当前第二批扩展固定为四个方向：

1. HBM / 高性能存储代理组；
2. 先进封装组；
3. 算力设备组；
4. 失败样本 / 阴影样本组。

固定优先顺序也已经明确为：

1. 澜起科技、佰维存储（已完成）；
2. 长电科技、通富微电（已完成）；
3. 金盘科技、科华数据（下一步）；
4. 鸿博股份、中贝通信。

### 判断

这份规划的核心价值，不是多列几个热门公司，而是把当前最缺的四类机制角色补上：

1. 稀缺算力利润链（已由 HBM_高性能存储代理组补上首轮样本）；
2. 先进封装 CapEx 审计链（已由先进封装组补上首轮样本）；
3. 算力设备订单滞后兑现链；
4. 末端拥挤与影子资产反证链。

### 待验证点

1. A 股缺少纯 HBM 龙头，因此必须承认“代理样本”边界；
2. 先进封装组应重点审计 CapEx、验证节奏和现金流，而不是只写工艺叙事；
3. 算力设备组要重点审计合同负债、回款、应收和 OCF，而不是只写订单公告；
4. 失败样本是研究角色，不是提前宣判；
5. 北向最新个股统计恢复前，资金边界仍以公募和成交结构为主。
