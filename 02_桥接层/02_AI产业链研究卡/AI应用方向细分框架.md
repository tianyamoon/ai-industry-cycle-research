> 所属模块：02_桥接层/02_AI产业链研究卡
> 生成日期：`2026-07-04`
> 研究定位：把此前分散在 `应用成熟_生态验证组`、`应用继续验证_企业软件组`、`AI产业链下游研究_近三个月跟踪.md` 里的应用层内容，压成可持续维护的统一细分框架。

## 一句话结论

AI 应用层不能再用“下游应用”一个总标签处理，也不能只用少数强样本代表全部应用方向；正确口径应分成两层：`强验证跟踪层`保持少而硬，`L0/L1 扩样候选池`必须足够宽。除 `办公协同`、`综合平台/MaaS`、`AI教育场景硬件入口`、`金融信息入口`、`企业软件平台`、`实施集成/垂类落地`、`AI原生创意工具`、`数据智能/知识服务`、`端侧 Agent`、`具身/机器人应用观察`、`AI玩具/陪伴硬件观察` 外，还要显式纳入 `医疗/AI制药`、`营销电商/外贸`、`内容/IP/游戏`、`工业/能源/交通/政务`、`数据/安全/版权治理`、`出海应用/海外软件映射` 等候选池；当前进入经营审计的仍只是少数样本，扩样池只解决“漏样本”，不自动解决“兑现强度”。

## 一、为什么要单独立应用框架

当前应用层已经出现三个现实问题：

1. `成熟度差异过大`
   - 金山办公、同花顺这种已有高频入口和成熟付费体系的样本，和用友、汉得、拓尔思这类还在平台转型与项目制过渡期的样本，不能再混写。
2. `商业模式差异过大`
   - 订阅制、License、项目制实施、Token 计费、广告/流量转化、企业协同平台，并不是同一条兑现链。
3. `市场最容易错把主题热度当经营兑现`
   - 尤其是 AI 眼镜、Agent、机器人应用、AI 玩具和行业大模型，最容易在新闻热度里被提前长期化。

因此，应用层后续必须和 `AIDC供配电`、`资源运营层` 一样，拥有固定的细分坐标、第一变量和滚动维护入口。

## 二、应用层统一细分坐标

| 细分 | 当前代表样本 | 当前阶段 | 第一变量 | 升级条件 | 最容易犯的错 |
|---|---|---|---|---|---|
| 办公协同平台 | 金山办公 | 成熟兑现层 | 付费渗透、WPS 365 放量、续费与客单价 | 企业付费、合同与现金流继续同步改善 | 把投资收益或表观利润放大成全部经营兑现 |
| 综合平台 / MaaS / 行业平台 | 科大讯飞 | 高投入验证层 | 合同转收入、MaaS 收入、开发者与现金流 | 收入增长快于费用扩张，扣非与 OCF 修复 | 把生态广度直接等同于利润质量 |
| AI教育场景硬件入口 | 视源股份 | 兑现验证层 | 教育终端装机、AI 备课 / 课堂反馈 / 数字基座覆盖、扣非和 OCF | 教育品牌收入、AI 教育产品付费或合同、合同负债和 OCF 同步改善 | 把硬件换机、控制部件增长或非经常性收益误写成 AI 教育兑现 |
| 金融信息入口 | 同花顺 | 成熟兑现层 | 活跃流量、产品转化、合同负债、经营杠杆 | 市场景气回落后仍保持收入与现金流韧性 | 把资本市场活跃度误写成纯 AI 兑现 |
| 企业软件平台 | 用友网络 | 继续验证层 | BIP 增长、AI 收入、签约客户、毛利率、OCF | AI 合同更快转收入，亏损与现金流同步修复 | 把大客户签约当成规模化复制 |
| 实施集成 / 垂类企业落地 | 汉得信息 | 继续验证层 | AI 收入占比、项目复购、扣非、回款 | 项目制收入转向更稳定的产品/服务复购 | 把“有落地项目”当成可持续高毛利模式 |
| AI 原生创意工具 | 万兴科技 | 继续验证层 | AI 原生收入、付费用户、Token、移动端占比 | 增长继续沉淀到利润与 OCF | 把用户增长直接等同于利润兑现 |
| 数据智能 / 知识服务转型 | 拓尔思 | 转型观察层 | 验收回款、Token/RaaS、收入恢复 | 收入恢复增长并出现更稳定的回款与扭亏 | 把远期模式创新直接折现进当前业绩 |
| 端侧 Agent / AI 眼镜 | 讯飞 AI 眼镜等观察样本 | 观察层 | 量产、订单、收入占比、订阅绑定 | 从展示/发布走向量产和持续收入 | 把发布会热度写成产业兑现 |
| 具身 / 机器人应用 | 观察池 | 观察层 | 量产、订单、收入占比、客户验证 | 正式订单和收入开始稳定披露 | 把映射交易和合作传闻写成主结论 |
| AI玩具 / 陪伴硬件 | 观察池 | 观察层 | 出货、渠道动销、退货率、复购、内容订阅 | 出货与复购稳定，出现内容或订阅变现 | 把社交传播热度写成持续消费兑现 |

## 三、应用层不是一条兑现曲线，而是六种模式

### 1. 平台订阅型

代表：`金山办公`

核心链条：

`高频办公入口 -> 协同渗透 -> 企业付费升级 -> 续费 / ARPU -> 现金流`

最该看：

1. 企业客户放量
2. 续费率
3. 合同负债
4. 经营现金流

### 2. 平台生态型

代表：`科大讯飞`

核心链条：

`模型与平台能力 -> 行业场景扩张 -> 合同 / 商机储备 -> MaaS / API 收入 -> 利润与现金流修复`

最该看：

1. 合同转收入速度
2. MaaS 收入
3. 扣非利润
4. 经营现金流

### 3. 高入口转化型

代表：`同花顺`

核心链条：

`高频金融入口 -> AI 提升产品转化 -> 合同负债 -> 收入与现金流杠杆`

最该看：

1. 景气回落后的收入韧性
2. 合同负债
3. 销售费用投产比

### 4. 企业转型/实施型

代表：`用友网络 / 汉得信息 / 万兴科技 / 拓尔思`

核心链条：

`AI 产品矩阵 / 项目交付 -> 收入占比 -> 毛利率 -> 扣非利润 -> OCF`

最该看：

1. AI 收入占比
2. 回款
3. 扣非利润
4. OCF

### 5. 教育硬件入口型

代表：`视源股份`

核心链条：

`教室终端装机 -> AI 备课 / AI 授课 / 课堂反馈 / 数字基座 -> 学校与区域覆盖 -> 合同负债 / 教育品牌收入 -> 扣非利润与 OCF`

最该看：

1. 教育品牌收入
2. AI 教育产品付费、合同或项目数
3. 合同负债
4. OCF、存货和备货压力

### 6. 前沿终端陪伴型

代表：`端侧 Agent / 机器人应用 / AI玩具`

核心链条：

`发布与体验反馈 -> 量产 / 出货 -> 复购或订阅绑定 -> 收入占比 -> 售后 / 内容 / 服务延伸`

最该看：

1. 量产和交付
2. 退货率与复购
3. 订阅或内容付费
4. 收入占比

## 四、当前优先级排序

### 0. 覆盖优先：候选池必须先变宽

应用方向的覆盖池不应少于 `办公 / 企业软件 / 金融 / 医疗 / 教育 / 营销电商 / 内容IP / 数据安全版权 / 视觉SDK / 端侧入口 / AI玩具` 十一组。这里的“变宽”只解决防漏，不改变正式打分；每个候选都必须先回源到付费、合同、续费、收入、扣非、OCF、出货或订阅，才能进入下列优先级。

### 1. 第一优先：已经进入经营兑现验证的应用龙头

1. `金山办公`
2. `科大讯飞`
3. `同花顺`
4. `视源股份`

原因：

1. 它们已经不只是“有产品”，而是有较清晰的付费、合同、现金流、经营杠杆或公告级利润验证线索。
2. 也是最适合进入结果层持续跟踪的应用样本。

### 2. 第二优先：企业软件与垂类应用的继续验证

1. `用友网络`
2. `汉得信息`
3. `万兴科技`
4. `拓尔思`

原因：

1. 这一层最能回答“AI 应用到底能否从产品走向收入与现金流”。
2. 也是当前应用层最大的研究缺口。

### 3. 第三优先：端侧 Agent、机器人与 AI玩具观察

原因：

1. 市场热度高，但正式经营证据仍弱。
2. 只能保留结构，不宜提前上收为主线结论。

## 七、一句话固定口径

> AI 应用层当前不是“全面开花”，而是“办公协同、综合平台、金融入口和 AI 教育硬件入口先进入经营质量审计，企业软件和垂类应用继续验证，端侧 Agent、机器人与 AI玩具仍停留在观察层”。
