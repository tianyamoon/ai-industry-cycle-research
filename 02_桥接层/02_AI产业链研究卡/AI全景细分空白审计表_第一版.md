## 一句话结论

当前已经建立全景细分样本宇宙入口，并新增 `AI全产业链财经网站样本资料库_第一版.md` 作为首批 `170+` 候选和 `2026-07-05` 页面化采集批次的 L0/L1 防漏池；本批东财 / 同花顺页面源按代码去重达到 `1742` 个标的，韭研公社补入 `72` 条帖子和 `93` 个社区股票标签，但还没有完成 `1000+ L0 / 500+ L1 / 240+ L2 / 120+ S3-S4` 的证据验收；最主要的空白仍集中在模型层、数据层、工具链、安全治理、消费入口、AI玩具、车端AI和阴影反证样本的连续变量。

覆盖状态、终端缺口、MCP 状态和回源批次一律以 `AI全景细分样本宇宙总表_第一版.md` 为唯一依据；本表只负责把“哪里没补到、为什么还不能引用、下一步该补什么”写清楚。

## 一、审计等级

| 状态 | 定义 | 允许结论 |
|---|---|---|
| `0样本` | 细分尚未登记任何样本 | 不允许引用 |
| `仅L0` | 只有候选或关注对象 | 只能防漏 |
| `缺L2` | 有结构定位，但缺可跟踪变量 | 不能进入组比较 |
| `缺S3-S4` | 有变量，但缺正式来源、连续验证或证伪点 | 谨慎用于桥接层，不进结果层强结论 |
| `可引用` | 已达到 S3-S4 | 可支撑桥接层或结果层 |

补充审计规则：

1. 若某细分样本主要来自财经网站、研报摘要、社区转载或门户聚合，且尚未回到公告、年报、IR、产品页或正式数据库，最多判到 `仅L0` 或 `缺L2`。
2. `终端待导入` 不阻塞样本先入库，但未导入的终端字段不得视为已完成补证。
3. `MCP待核` 或 `待接入` 只代表行情层占位，不得单独推动样本升到 `L2 / S3-S4`。

## 二、主层级空白审计

| 主层级 | 当前覆盖状态 | 主要缺口 | 下一步动作 |
|---|---|---|---|
| 计算芯片 | `缺S3-S4` | A股端侧SoC、车载AI芯片、DPU/SmartNIC 的经营变量密度不足 | 先补海外定义者，再补 A股映射的收入、客户和订单口径 |
| 芯片设计基础 | `缺L2` | EDA、IP、验证服务多数仍停留在产业位置，缺 AI 相关收入和客户变量 | 补年报、IR、产品页，区分国产替代与 AI 增量 |
| 晶圆制造与材料 | `缺S3-S4` | 强样本集中在设备/材料龙头，pellicle、掩模 blank、前驱体、InP/GaAs 二线仍薄 | 按日本材料、韩国测试、A股材料平台三条线继续补证 |
| 存储链 | `缺S3-S4` | HBM 本体强、A股代理和测试配套仍需收入拆分 | 补 HBM 价格、供给、客户绑定和 A股配套变量 |
| 先进封装 | `缺S3-S4` | glass、panel-level、hybrid bonding 多数仍是观察层 | 区分现实主线 ABF/FCBGA 与期权路线 glass/TGV |
| 高速互联 | `缺S3-S4` | 光模块强样本多，CPO、硅光、LPO、coherent optics 的连续经营验证不足 | 建路线之争表，避免把替代路线直接写成兑现 |
| 整机系统 | `缺L2` | ODM/JDM、机柜、背板、线缆、维保的 AI 纯度待核 | 补订单、库存、ASP、OCF |
| 数据中心与基础设施 | `缺S3-S4` | AIDC收入拆分、上架率、利用率、融资成本仍缺连续口径 | 与资源运营评分表和 H1 模板联动 |
| 供配电与温控 | `缺S3-S4` | 低 AI 纯度电力设备旁证多，核心 AIDC 订单纯度待核 | 按三路径判定表回写经营闭环 |
| 云与算力运营 | `缺L2` | 海外云 CapEx 强，本土算力租赁回款、利用率和价格变量薄；订单驱动型算力租赁、云服务转型和跨界算力样本此前未形成统一 L0/L1 池，说明防漏仍不足 | 拆云厂、租赁、MaaS、推理平台四类；对合同交付、起租、回款、融资成本、利用率和 OCF 做分组审计 |
| 模型层 | `缺L2` | 私有模型多，财务与调用量披露弱；评测、推理引擎、编译优化样本不足 | 先保留 S0-S1，补产品、API、客户和成本变量 |
| 数据层 | `缺L2` | 版权数据、合成数据、数据合规、隐私计算样本弱 | 建数据供给、处理、合规三张子表 |
| 工具链与基础软件 | `缺L2` | 开源项目多，收入与商业化弱；国内映射不足 | 区分开源基础设施、SaaS工具、企业中台 |
| 安全合规治理 | `缺L2` | AI安全新样本多为私有公司或产品页，A股对应弱 | 先建观察池，不上收强结论 |
| 应用平台 | `缺S3-S4` | 产品发布多，AI收入占比、续费、扣非和 OCF 仍不足 | 与应用方向滚动跟踪总表联动 |
| 消费入口 | `缺L2` | 海外产品强，国内收入和留存证据弱 | 先看用户、付费、留存、获客成本 |
| 终端与具身智能 | `缺L2` | 机器人、AI眼镜、AI耳机、灵巧手等多为量产早期 | 用 BOM、出货、客户、订单和退货率审计 |
| AI玩具与陪伴硬件 | `缺L2` | 私有品牌多，正式财务弱，订阅与复购待核 | 保留观察层，补出货、订阅、渠道和售后 |
| 车端AI | `缺L2` | 车端模型、数据闭环和域控变量需要与新能源母体交叉 | 补销量、软硬件收费、传感器、车端芯片 |
| 阴影与反证 | `缺S3-S4` | 数量仍不足，且未覆盖所有主层级；跨界算力转型、大额框架合同和重资产融资样本仍有漏记 | 单独扩到 50+，覆盖硬件、软件、基建、应用、终端；把跨界算力转型样本作为反证观察池，持续看履约和现金流 |

## 三、当前最优先补的空白细分

| 优先级 | 细分 | 原因 | 最小补证动作 |
|---|---|---|---|
| 1 | 数据层 / 安全治理 | 当前最容易被硬件链挤压，且披露弱 | 先补 L0-L1 样本池和第一变量 |
| 2 | 模型层 / 推理部署 / 编译优化 | 直接影响云 CapEx 到推理 Opex 的转化 | 补海外产品、开源项目和商业化变量 |
| 3 | 应用平台 / 消费入口 | 决定 AI 是否从 CapEx 走向收入兑现 | 补付费、续费、合同、OCF |
| 4 | 终端 / 机器人 / AI玩具 | 与新能源母体差异最大，样本仍薄 | 补出货、BOM、ARPU、退货率 |
| 5 | 阴影反证 | 关系到幸存者偏差和题材外溢审计 | 从每个主层级各补至少 2-3 个反证样本 |
| 6 | 算力租赁 / 跨界算力转型 | 传统 IDC / AIDC 样本不能覆盖订单驱动型算力合同、云服务转型和重资产融资样本 | 先按样本组补公告原文、合同金额、服务期、交付起租、应收回款、融资成本、利用率和 OCF |
| 7 | 财经网站 L0/L1 候选池批量回源 | 新增资料库已补首批 `170+` 线索，并追加 `1742` 个东财 / 同花顺页面可见代码和 `93` 个韭研公社社区股票标签；但大多仍停留在页面概念、财经媒体、研报摘录或社区线索 | 按算力租赁、Agent应用、AI玩具/眼镜、设备材料、PCB/CPO、人形机器人、AI语料七批回源到公告、年报、IR和财务字段 |

## 四、验收纪律

1. 所有状态判定先看 `AI全景细分样本宇宙总表_第一版.md`，再看本表，不允许反向在本表里“口头补完成”。
2. 全景图每个逗号分隔细分都必须至少有 `L0` 记录。
3. 每个主层级至少要有 `L2` 样本，不能只有观察名单。
4. 未达到 `S3-S4` 的细分，不得进入结果层强结论。
5. 如果某细分只有海外样本，应明确写成“海外变量定义层”，不得替代 A股验证。
6. 若细分长期只有传闻、发布会、产品页，应维持 `S0-S1`。
7. 如果某主层级只有财经网站、研报摘要、社区转载和门户转载样本，不能判为 `可引用`。
8. 终端缺口必须明确写 `待导入`，不能用“终端看过”代替回写。
9. MCP 出现空成功、字段漂移或本轮尚未接入时，行情字段只作占位，不得直接抬升样本等级。
