# AI算力基建大组扩展观察池_第一版

> 所属模块：02_桥接层/02_AI产业链研究卡  
> 生成日期：2026-07-04  
> 研究定位：把窄口径 `AIDC供配电` 三路径样本，扩展为 `AI算力基建大组` 的统一观察池，用于判断资金和经营验证是否从核心供配电扩散到液冷温控、IDC / AIDC 运营和电力设备旁证层。  
> 使用边界：本文是桥接层观察池，不是买入清单；扩大样本范围不等于扩大结论确定性，低 AI 纯度样本只能作为旁证，不能直接替代核心供配电样本。

---

## 一句话结论

`科华数据 / 金盘科技 / 科士达` 三只样本适合做 `AIDC供配电` 的核心三路径锚，但样本太小，不能单独代表完整 AI 算力基建板块。后续应把观察范围扩成：

`核心供配电 -> 电力设备旁证 -> 液冷 / 温控 -> IDC / AIDC运营`

其中，窄组三只继续负责选股和经营闭环验证；大组观察池负责判断是否真的出现 `AI算力基建` 组级共振。

---

## 一、为什么要扩组

### 事实

1. 结果层 `AIDC供配电三路径接力判定表_第一版.md` 已经把 `科华数据 / 金盘科技 / 科士达` 拆成三条路径：
   - `订单型高弹性设备`
   - `平台型智算供电基础设施`
   - `标准化稳态兑现`
2. 但三只样本只能回答 `供配电内部谁先拿到订单 / 收入 / OCF 闭环`，不能回答完整基建链是否启动。
3. 现有库中已经有 `液冷_散热二线弹性组`、`数据中心运营_算力租赁组` 和 `AI产业链候选标的池_第一版.md`，说明基础设施协同层本来就不只一组样本。
4. 最新周更口径仍显示：`AIDC供配电` 未确认接力，`液冷` 未确认组级扩散，`IDC / AIDC运营` 仍是修复观察。

### 判断

扩组的目的不是降低门槛，而是提高判断质量：

1. 如果只有三只供配电样本转强，可能只是小组内补涨；
2. 如果供配电、液冷、IDC运营、电力设备旁证同时改善，才更像 AI 算力基建大组共振；
3. 如果大组扩散但经营层没有 `收入 / 利润 / OCF / 回款` 验证，则更可能是题材外溢，而不是接力确认。

## 三、窄组和大组的分工

### 1. 窄组：用于选股和经营闭环

窄组仍然是：

`科华数据 / 金盘科技 / 科士达`

它解决三个问题：

1. `金盘科技` 的 `订单 -> 收入 -> OCF` 能否闭环；
2. `科华数据` 的 `分部收入 -> 交付效率 -> OCF` 能否继续强化；
3. `科士达` 的 `数据中心核心业务 -> AI纯度 -> OCF持续性` 能否进一步清晰。

### 2. 大组：用于判断是否成为基建共振

大组观察池解决四个问题：

1. 供配电是否不再只是三只个股的局部修复；
2. 液冷 / 温控是否从单点强势扩散到组级行情；
3. IDC / AIDC运营是否从退潮修复变成真实运营改善；
4. 电力设备旁证是否出现与 AIDC 电力需求相关的订单、交付或收入拆分。

## 四、组级升级标准

### 1. 观察层

默认当前状态仍应写为：

`经营有线索，行情未确认，扩组后仍是观察池。`

触发条件：

1. 供配电三只样本尚未形成 `20日转正 + 60日超过30%`；
2. 液冷仍偏单点强势；
3. IDC / AIDC运营仍缺上架率、利用率和 OCF 连续改善。

### 2. 初步修复层

可以上调为 `基建修复观察` 的条件：

1. 核心供配电组至少两只 `20日涨幅转正`；
2. 液冷 / 温控组不再只有单点强势，至少两只同步改善；
3. IDC / AIDC运营层至少出现一只经营层强验证样本；
4. 新增公告或中报至少补出 `订单 / 收入 / OCF` 中的一项正式升级。

### 3. 接力候选层

可以上调为 `基建接力候选` 的条件：

1. 核心供配电组 `60日组均涨幅` 接近或超过 `30%`；
2. 大组至少三个子层中有两个子层同步转强；
3. 至少一条路径形成 `订单 / 收入 / OCF` 连续闭环；
4. 另外一条路径不拖后腿，证明不是孤立个股行情。

### 4. 主线确认层

只有同时满足行情和经营两类条件，才可写成 `AI算力基建主线确认`：

1. 行情层：供配电、液冷、IDC运营或电力设备旁证中，至少三个子层出现组级强度；
2. 经营层：核心供配电或液冷 / 温控里至少两家公司出现收入、利润、OCF 同步改善；
3. 证伪层：应收、存货、合同负债、在建工程、融资成本没有明显恶化；
4. 资金层：不只是题材换手上升，而是机构持仓代理或成交结构同步改善。

## 五、变量表

| 变量 | 频率 | 对应层级 | 当前用法 | 加分条件 | 减分条件 |
|---|---|---|---|---|---|
| 数据中心订单 | 季度 / 公告 | 核心供配电、电力设备旁证 | 验证需求是否真实进入设备链 | 订单继续高增且客户 / 项目口径清楚 | 只重复订单口径，收入不跟 |
| 数据中心收入拆分 | 季度 / 年报 | 核心供配电、IDC运营 | 验证订单是否转收入 | 分部收入继续增长，AI纯度更清楚 | 收入口径模糊或被新能源 / 传统业务掩盖 |
| OCF / 回款 | 季度 | 全部层级 | 判断利润质量 | OCF与利润同步改善 | 利润增长但 OCF 持续走弱 |
| 上架率 / 利用率 | 季度 / 年报 | IDC / AIDC运营 | 判断资源运营质量 | 上架率和利用率提升，租赁价格稳定 | 折旧和融资压力先于收入改善 |
| 液冷订单与毛利率 | 季度 / 公告 | 液冷 / 温控 | 判断散热需求是否扩散 | 订单转收入，毛利率改善，回款同步 | 订单强但收入确认和现金流错位 |
| 应收 / 存货 / 合同负债 | 季度 | 项目制设备与液冷 | 判断项目制风险 | 合同负债抬升伴随收入和回款改善 | 备货重、应收升、OCF不改善 |
| 20日 / 60日组均涨幅 | 周度 | 全部层级 | 判断行情是否共振 | 20日转正，60日接近或超过30% | 只有单只强，组均仍弱 |
| 成交额 / 换手 | 周度 | 全部层级 | 判断资金参与 | 成交放大但未极端拥挤 | 高换手无经营增量 |

## 六、当前正式口径

截至本版，扩组后的正式口径应写为：

1. `AIDC供配电` 三只窄组仍是核心经营锚；
2. `AI算力基建大组` 已补成观察池，但尚未形成主线确认；
3. `液冷 / 温控`、`IDC / AIDC运营` 与 `电力设备旁证` 只能用于判断共振，不应直接替代三路径经营闭环；
4. 后续若大组走强，必须回到 `订单 / 收入 / OCF / 回款 / 利用率 / AI纯度` 做证据审计。

## 八、下一步最优先补证

1. 给大组观察池补一张周度行情表：`5日 / 20日 / 60日 / YTD / 距60日高点 / 组内上涨家数`。
2. 对电力设备旁证组逐只确认 `AIDC / 数据中心 / 海外电力设备` 收入口径，不能只用电网设备总增长替代。
3. 对液冷 / 温控组补 `液冷订单占比 / 收入确认 / 毛利率 / OCF`。
4. 对 IDC / AIDC运营组补 `上架率 / 利用率 / 租赁价格 / 折旧 / 融资成本`。
5. 若后续周更继续显示大组不共振，则维持 `观察池`，不得升级为 `基建接力`。
