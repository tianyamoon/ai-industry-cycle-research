# AI应用方向滚动跟踪总表
> 所属模块：02_桥接层/02_AI产业链研究卡
> 生成日期：`2026-07-04`
> 研究定位：把 AI 应用层从“有方向稿、有单卡、但缺统一维护入口”推进到可滚动维护的桥接层总表。
> 使用边界：本表优先使用已完成的单卡、组比较、下游方向稿与结果层估值总表；若无新增正式公告，不强行迁移状态。
> 最近更新：`2026-07-12`，观察池口径已升级为全产业链 A 股总市值 `100亿元` 强制观察；本表只承接其中应用相关的状态迁移，不再维护局部市值规则。

---

## 一句话结论

当前 AI 应用方向已经具备第一轮统一维护条件，但正式打分样本不是应用层覆盖上限：`金山办公 / 同花顺 / 视源股份 / 科大讯飞 / 用友网络` 属应用成熟与继续验证层，`万兴科技` 属 AI原生创意工具观察层，`拓尔思` 属数据智能反证层；与此同时，必须把 `医疗AI / AI制药 / 营销电商 / 内容IP / 工业政务 / 数据安全版权 / 端侧Agent / AI玩具` 纳入 L0/L1 候选池。应用层只是全产业链百亿观察池的一个子集，不能再用应用层市值规则代替全局覆盖。后续滚动维护必须回到 `付费 / 合同 / AI收入占比 / 扣非利润 / 经营现金流`，而不是继续把应用层写成一组主题名字。

## 一、统一维护口径

### 1. 三项统一评分

1. `商业化验证分`
   - 看真实付费、合同、AI 收入、续费或 Token 模式是否成立。
2. `报表传导分`
   - 看收入、扣非利润和 OCF 是否开始同步传导。
3. `风险审计分`
   - 看获客成本、费用率、回款、合同负债、盈利质量与估值久期是否可控。

### 2. 单项分值

1. `3分`
   - 连续验证较清楚。
2. `2分`
   - 已验证但仍需继续跟踪。
3. `1分`
   - 逻辑未破，但经营传导仍弱。
4. `0分`
   - 仍偏题材或进入明显证伪审计。

### 3. 总分对应状态

1. `8-9分`
   - 接近成熟兑现样本。
2. `6-7分`
   - 继续验证，值得优先维护。
3. `4-5分`
   - 转型 / 观察，不宜提前长期化。
4. `0-3分`
   - 高波动主题位，优先做反证。

## 三、当前组级判断

### 1. 哪些应用已经进入成熟兑现验证

当前更接近成熟兑现验证的是：

1. `金山办公`
2. `同花顺`
3. `视源股份`

原因：

1. 它们都能把 AI 放回既有高频入口、协同场景或教育场景中解释经营结果。
2. 不再只是产品热度，而是出现了更清楚的收入、合同负债或现金流信号。
3. 视源股份已出现公告级 H1 扣非高增预告和正式披露的 AI 教育产品覆盖数据，但需要保守扣除非经常性收益和硬件备货扰动。

### 2. 哪些应用仍属于“逻辑成立，但报表仍在验证”

1. `科大讯飞`
2. `用友网络`
3. `汉得信息`

原因：

1. 平台与场景逻辑清晰。
2. 但利润和现金流仍未完全跟上。

### 3. 哪些应用还不能提前长期化

1. `万兴科技`
2. `拓尔思`
3. `机器人 / 具身应用观察池`
4. `AI玩具 / 陪伴硬件观察池`

原因：

1. 模式切换与收入结构升级方向存在。
2. 但报表传导和 OCF 仍偏弱。
