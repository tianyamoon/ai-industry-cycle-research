## 一、一句话结论

> 当前最该继续推进的，不是再扩更多主线名单，而是把 `资源运营层` 和 `液冷二线` 放到同一张经营质量台账里审计。现阶段最接近“经营兑现锚点”的是 `润泽科技`、`奥飞数据` 与 `宝信软件`，最稳的现金流锚点是 `数据港`，而 `光环新网`、`首都在线`、`申菱环境`、`高澜股份` 更需要沿着 `收入确认 / OCF / 折旧或存货 / 融资与回款` 的链条继续验证。

## 二、评分原则

### 1. 三项评分结构

每个样本统一按三项评分：

1. `事实升级分`
   - 看订单、交付、上架率、利用率、业务占比、客户验证是否继续升级。
2. `报表传导分`
   - 看收入、利润和 OCF 是否开始同步传导。
3. `风险审计分`
   - 看折旧、融资、长期应付款、存货、回款、诉讼或项目制波动是否仍可控。

### 2. 单项分值

每项默认按 `0-3 分` 处理：

1. `3 分`
   - 已形成持续验证。
2. `2 分`
   - 已验证但仍需继续跟踪。
3. `1 分`
   - 逻辑未破，但经营质量开始承压。
4. `0 分`
   - 已进入高波动、待降级或待证伪状态。

### 3. 总分对应状态

1. `8-9 分`
   - 接近正式经营兑现样本。
2. `6-7 分`
   - 经营验证中，值得优先维护。
3. `4-5 分`
   - 继续观察，不宜提前长期化。
4. `0-3 分`
   - 高波动风险位，优先做反证审计。

## 五、资源运营层滚动台账

### 1. 当前组级判断

资源运营层当前最重要的不是“谁有更多机柜”，而是：

`谁能把资源投放 -> 收入口径 -> OCF -> 折旧 -> 融资结构` 解释成一条自洽链条。

### 2. 六个样本的优先维护点

| 公司 | 当前最该盯的变量 | 当前最敏感反证点 |
|---|---|---|
| 润泽科技 | AIDC 收入占比、上架率、利用率、OCF 持续性 | 长期应付款、折旧和资本循环压力 |
| 数据港 | 智算收入导入、利用率、稳态 OCF | 智算导入长期停留在口径层 |
| 奥飞数据 | IDC 与算力服务收入、OCF、在建转固节奏 | 固定资产、借款、长期应付款继续扩张 |
| 光环新网 | 上架率、租赁价格、IDC 毛利率、回款 | 上架率低位停滞、租赁价格下行、OCF 继续偏弱 |
| 宝信软件 | 宝之云交付、算力中心进度、OCF 持续性 | 资源运营拆分不足、回报率长期不清晰 |
| 首都在线 | 智算云收入占比、OCF 修复、融资与诉讼变化 | OCF 再走弱、融资租赁和借款压力放大 |

### 3. 当前统一审计口径

资源运营层统一回到以下变量：

1. 上架率
2. 利用率
3. GPU / 算力租赁价格
4. AIDC / 智算收入占比
5. OCF
6. 折旧与摊销压力
7. 长期应付款 / 有息负债 / 融资结构
8. CapEx 与资本循环效率

## 六、液冷二线滚动台账

### 1. 当前组级判断

液冷二线当前不能再只看“液冷方向对不对”，而要看：

`订单质量 -> 收入确认 -> 毛利率 -> 回款 / 存货 / OCF`

这条链路能否闭环。

### 2. 两个样本的优先维护点

| 公司 | 当前最该盯的变量 | 当前最敏感反证点 |
|---|---|---|
| 申菱环境 | 新增订单、数据服务收入、项目回款、OCF | 订单强但收入确认和 OCF 继续错位 |
| 高澜股份 | 毛利率、扣非利润、存货、OCF | 经营修复只停留在利润表，现金流与存货不跟 |

### 3. 当前统一审计口径

液冷二线统一回到以下变量：

1. 订单质量
2. 收入确认节奏
3. 毛利率
4. 存货
5. 回款
6. OCF

## 七、状态迁移规则

### 1. 升级条件

样本只有在以下三条同时改善时，才从“继续观察”升级到“经营验证中”或“接近经营兑现”：

1. 新披露事实继续升级；
2. 收入、利润和 OCF 至少两项同步强化；
3. 折旧、融资、存货、回款等审计项没有明显恶化。

### 2. 降级条件

若出现以下任一情况，应优先考虑降级或转入反证审计：

1. 事实继续讲故事，但报表传导停滞；
2. OCF 连续走弱，同时资产负债表压力放大；
3. 项目制或租赁型样本的回款、折旧、融资或诉讼风险显著放大。

### 3. 本表的纪律

1. 不把缺连续披露的数据硬写成结论。
2. 不把题材热度写成经营兑现。
3. 不把资源运营层和液冷二线混写成统一高景气主线。
