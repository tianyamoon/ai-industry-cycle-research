## 二、桥接层固定坐标

### 1. 核心算力资产层
1. GPU / ASIC
2. HBM / 高性能存储
3. 先进封装
4. 光模块 / 高速互联

### 2. 先进制造上游卡点与核心弹性层
1. 晶圆制造原材料与重复消耗型工艺耗材
2. 硅片、靶材、高纯石英等供给安全材料
3. 精密零部件、高纯流体、真空、陶瓷 / 石英件与制程系统
4. 先进封装、高速互联、AIDC 的上游材料 / 核心部件路由
5. 资源供给与现场气体只作为供给约束和经营拆分观察，不替代关键卡点结论

### 3. 整机与平台承接层
1. AI 服务器
2. 交换机 / 网络系统
3. 国产算力平台
4. 算力设备组

### 4. 数据中心与算力运营层
1. IDC / AIDC
2. 算力租赁
3. 算力调度
4. 资源型重资产平台

### 5. 基础设施协同层
1. 电力设备
2. 液冷 / 温控
3. PCB / CCL / 连接器

### 6. 数据、工具与安全治理层
1. 训练数据 / 数据处理
2. 可信数据空间 / 数据治理
3. AI 安全产品
4. 测评 / 审计 / 治理平台

### 7. 全球映射与海外变量传导层
1. 海外 GPU / HBM / 云 CapEx 主链
2. 海外芯片制造与材料深链
3. 海外模型平台与企业软件主链
4. 海外终端 / 机器人 / AI玩具主链
5. A 股跟随 / 补位 / 配套 / 主题映射审计

### 8. 应用与终端兑现层
1. 办公软件
2. 企业软件
3. 行业平台
4. 端侧 Agent / AI 眼镜
5. 机器人 / 具身应用
6. AI玩具 / 陪伴硬件

### 9. 影子资产与前沿观察层
1. 失败样本 / 阴影样本
2. 玻璃基板 / TGV
3. 新型工艺观察池

## 四、桥接层当前职责

1. 定义样本位置，不替代结果层正式总索引。
2. 统一样本命名和分组口径，避免同一公司在不同文档里换角色。
3. 给每张研究卡提供统一问题框架：位置、买方、钱包位置、第一变量、先看什么、后放弃什么、映射谁。
4. 把新增样本优先补到空白节点，而不是追逐最新热词；上游扩样必须先区分原材料、核心零部件、制程系统与资源供给，不能再把它们统称为“气体”或“上游平台”。
5. 对 AI 算力基建，先用扩展观察池判断组级共振，再回到窄口径三路径表判断经营闭环，避免把低 AI 纯度电力设备旁证直接写成主线结论。
6. 对样本扩容，先回到 `AI样本扩容与偏差控制主表.md` 标注 `S0-S4` 等级，再决定是否进入组比较或结果层。
7. 对财经网站和社区线索，不能只统计题材成份股数量；必须先进入 `AI全产业链标的地图与跟踪总表.md`，按产业位置、样本角色、第一变量、跟踪频率和证伪点重排，再决定是否回写单卡或结果层。

## 五、当前显性缺口

1. 样本代表性不足已升为偏差控制项，需把约 31 个 A 股主样本扩到 96+ 有效样本位。
2. 数据层和安全治理层方向研究已补，但正式单卡和连续财务验证仍少。
3. 终端、机器人与 AI玩具虽已纳入统一坐标，但正式样本组仍偏少。
4. 海外供应链变量与 A 股映射已有独立入口，但周度连续回写仍需持续跟踪。
5. 全球制造深链虽然已有独立入口，但 `日本材料簇 / 韩国 HBM 配套簇 / InP 光器件簇 / 基板路线簇` 的 A 股回落映射仍需继续细化；制造上游的精密部件、真空、高纯流体、石英 / 陶瓷件样本也需补产品、客户、收入和 OCF 回源。
6. AI算力基建大组已经补出扩展观察池，但电力设备旁证组、液冷温控组和 IDC / AIDC运营组仍需继续补 `AI纯度 / 订单拆分 / 上架率 / OCF`。
7. 前沿观察池中的若干主题尚未升级为正式样本。
