# AI应用方向滚动跟踪总表_第一版
> 所属模块：02_桥接层/02_AI产业链研究卡  
> 生成日期：`2026-07-04`  
> 研究定位：把 AI 应用层从“有方向稿、有单卡、但缺统一维护入口”推进到可滚动维护的桥接层总表。  
> 使用边界：本表优先使用已完成的单卡、组比较、下游方向稿与结果层估值总表；若无新增正式公告，不强行迁移状态。  

---

## 一句话结论

当前 AI 应用方向已经具备第一轮统一维护条件：`金山办公 / 科大讯飞 / 同花顺` 属成熟兑现层，`用友网络 / 汉得信息 / 万兴科技` 属继续验证层，`拓尔思` 属转型观察层；与此同时，`端侧 Agent / 机器人 / AI玩具` 已纳入统一观察池。后续滚动维护必须回到 `付费 / 合同 / AI收入占比 / 扣非利润 / OCF`，而不是继续把应用层写成一组主题名字。

---

## 一、统一维护口径

### 1. 三项统一评分

1. `商业化验证分`
   - 看真实付费、合同、AI 收入、续费或 Token 模式是否成立。
2. `报表传导分`
   - 看收入、扣非利润和 OCF 是否开始同步传导。
3. `风险审计分`
   - 看获客成本、费用率、回款、合同负债、盈利质量与估值久期是否可控。

### 2. 单项分值

1. `3分`
   - 连续验证较清楚。
2. `2分`
   - 已验证但仍需继续跟踪。
3. `1分`
   - 逻辑未破，但经营传导仍弱。
4. `0分`
   - 仍偏题材或进入明显证伪审计。

### 3. 总分对应状态

1. `8-9分`
   - 接近成熟兑现样本。
2. `6-7分`
   - 继续验证，值得优先维护。
3. `4-5分`
   - 转型 / 观察，不宜提前长期化。
4. `0-3分`
   - 高波动主题位，优先做反证。

## 三、当前组级判断

### 1. 哪些应用已经进入成熟兑现验证

当前更接近成熟兑现验证的是：

1. `金山办公`
2. `同花顺`

原因：

1. 它们都能把 AI 放回既有高频入口或协同场景中解释经营结果。
2. 不再只是产品热度，而是出现了更清楚的收入、合同负债或现金流信号。

### 2. 哪些应用仍属于“逻辑成立，但报表仍在验证”

1. `科大讯飞`
2. `用友网络`
3. `汉得信息`

原因：

1. 平台与场景逻辑清晰。
2. 但利润和现金流仍未完全跟上。

### 3. 哪些应用还不能提前长期化

1. `万兴科技`
2. `拓尔思`
3. `机器人 / 具身应用观察池`
4. `AI玩具 / 陪伴硬件观察池`

原因：

1. 模式切换与收入结构升级方向存在。
2. 但报表传导和 OCF 仍偏弱。

## 五、维护纪律

1. 不把 `AI产品发布` 直接写成 `商业化成立`。
2. 不把 `合同签约` 直接写成 `收入和现金流兑现`。
3. 不把 `用户增长` 直接写成 `盈利质量提升`。
4. 应用层任何上调，至少要看到 `付费/合同` 与 `报表/现金流` 中的一条同步强化。
