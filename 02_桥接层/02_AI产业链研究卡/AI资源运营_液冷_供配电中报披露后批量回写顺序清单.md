## 三、批量回写顺序

### 1. 先锁正式来源

只接受以下来源进入本轮正式回写：

1. 半年报、业绩预告、业绩快报；
2. 投资者关系活动记录表、业绩说明会记录；
3. 上交所 / 深交所 / 巨潮资讯正式公告；
4. 公司官网同步披露材料。

如果只有媒体二次转述：

1. 先记到临时待核，不进正式表；
2. 等正式链接补齐后再回写。

### 2. 先填 `2026H1` 模板

固定顺序：

1. 先填披露日期；
2. 再填 `收入 / 归母 / OCF`；
3. 再填组内关键经营变量；
4. 若是 `AIDC供配电`，同步填 `三路径内部判断 / 是否触发接力升级`；
5. 最后填 `新总分 / 状态迁移 / 是否回写单卡`。

纪律：

1. 不允许先改分数、后补数据。
2. 对披露不足的字段保留 `待补 / 待核 / 待确认`。

### 3. 再回写统一评分表

固定更新位置：

1. `总表`
2. `截至当前日期的正式核验`
3. `后续回填模板`
4. `配套文件入口` 如需补新工具

回写时重点确认：

1. 总分是否迁移；
2. 状态是否迁移；
3. 组级一句话判断是否需要改写。

### 4. 再回写组内文件

## 四、批量回写执行表

| 顺序 | 动作 | 主要文件 | 触发条件 | 若不足以触发 |
|---|---|---|---|---|
| 1 | 锁正式来源 | 两张 `2026H1实填模板` | 找到正式公告 / IR / 半年报 | 保留 `待核` |
| 2 | 回填前值与中报字段 | 两张 `2026H1实填模板` | 披露了收入、利润、OCF 或关键变量 | 只填已披露数字 |
| 3 | 调整总分与状态 | 两张 `滚动评分表` | 至少两条验证链同步改善或恶化 | 维持原分数 |
| 4 | 组内回写 | 滚动验证清单 / 三卡比较 / 单卡 | 组内判断发生变化 | 只补来源或数字 |
| 5 | 结果层回写 | 风险信号 / 个股验证表 / 行情演绎表 | 上层口径被新事实改变 | 不回写 |
| 6 | 状态源回写 | 进度索引 / 模块索引 / 状态面板 / 交接包 | 维护入口或优先级改变 | 不回写 |

## 五、纪律补充

1. 不把单个样本的好中报写成整组行情确认。
2. 不把 `订单 / 上架率 / 机柜签约 / 在手订单` 直接等同于 `收入 / 利润 / OCF`。
3. 不为了赶进度删除 `待补 / 待核 / 待确认`。
4. 不绕过桥接层模板和评分表，直接改结果层结论。
5. `AIDC供配电` 若没有新的收盘交易日，不新起 dated 周更，先完成 `模板 -> 评分表 -> 三路径判定表` 的经营层回写，再等结果层行情字段更新。
