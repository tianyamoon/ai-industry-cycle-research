## 二、分标的执行要点

### 润泽科技

1. 优先盯中报、三季报中的 AIDC 收入占比与交付节奏。
2. 看 OCF 是否继续和收入扩张同步，而不是只剩利润表强。
3. 看长期应付款、融资结构和折旧压力是否开始快于经营兑现上升。

### 数据港

1. 优先盯成熟 IDC 托管业务的稳态 OCF 是否保持。
2. 看智算业务导入是否从“布局”升级到更明确的收入和客户口径。
3. 看利用率、上架率和折旧是否共同支撑转型效率，而不是只有概念升级。

### 奥飞数据

1. 优先盯 IDC 与算力服务收入能否继续带动 OCF。
2. 看固定资产、长期借款和长期应付款扩张是否快于经营兑现。
3. 看区域平台扩张能否继续转成更稳的客户结构和现金流。

### 光环新网

1. 优先盯上架率、租赁价格和 IDC 毛利率。
2. 看云业务承压是否继续拖累整体平台经营质量。
3. 看回款与 OCF 是否修复，否则大投放逻辑会被快速审计。

### 宝信软件

1. 优先盯宝之云项目持续交付和资源运营口径是否更细。
2. 看在建工程向固定资产转化后的回报率。
3. 看 OCF 是否继续强于利润波动。

### 首都在线

1. 优先盯智算云收入占比能否继续抬升。
2. 看 OCF 能否从 Q1 转负重新修复。
3. 看借款、融资租赁、在建工程和诉讼风险是否继续恶化。

## 三、组级观察原则

### 判断

1. 本组核心不是“算力租赁热度”，而是“资源平台经营质量”。
2. `润泽科技` 更适合按 `交付 -> 收入结构 -> OCF -> 融资与折旧审计` 路径跟踪。
3. `数据港` 更适合按 `稳态现金流 -> 智算导入 -> 转型效率` 路径跟踪。
4. `奥飞数据` 更适合按 `区域扩张 -> 收入与 OCF -> 资产负债表审计` 路径跟踪。
5. `光环新网` 更适合按 `机柜投放 -> 上架率 / 租赁价格 -> OCF 与毛利率审计` 路径跟踪。
6. `宝信软件` 更适合按 `项目交付 -> 园区运营 -> 回报率审计` 路径跟踪。
7. `首都在线` 更适合按 `智算收入占比 -> OCF -> 折旧 / 借款 / 诉讼审计` 路径跟踪。
8. 资源运营层必须把机柜、交付、现金流、折旧和负债放到同一张表里看，不能只看收入增速。

## 四、当前组级动作建议

1. `第一优先`：继续跟 `润泽科技` 的 AIDC 收入占比、交付节奏和 OCF。
2. `第二优先`：继续跟 `光环新网` 的上架率、租赁价格和 OCF。
3. `第三优先`：继续跟 `数据港` 与 `奥飞数据` 的现金流与智算导入效率。
4. `第四优先`：把 `宝信软件` 与 `首都在线` 作为质量补位和风险补位滚动维护。
5. `长期动作`：统一补这组最缺的连续口径：
   - 上架率
   - 利用率
   - GPU / 算力租赁价格
   - 长期应付款与折旧压力
