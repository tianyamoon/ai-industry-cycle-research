## 四、模板使用

### 1. 单卡模板
- 先写 `产业位置`
- 再写 `第一变量`
- 再写 `兑现顺序`
- 再写 `风险审计`
- 最后写 `新能源映射`

### 2. 组卡模板
- 先写组级一句话结论
- 再拆分核心样本、补位样本、旁证样本
- 再补组级验证清单
- 再给出当前优先级

### 3. 滚动清单模板
- 只保留下一条最关键验证数据
- 用 `加分信号 / 减分信号` 双栏
- 每次更新优先补正式公告和财报

## 五、半导体设备全景组的写法提醒

1. `长川科技` 写利润兑现主锚。
2. `盛美上海` 写订单强化补强位。
3. `京仪装备` 写 `平台扩线观察位` / `关键子系统配套层前排样本`。
4. `中微公司 / 北方华创 / 华峰测控 / 华兴源创 / 中科飞测 / 拓荆科技 / 芯源微` 写旁证扩展位。
5. `长电科技 / 通富微电 / 帝尔激光 / 莱宝高科 / 兴森科技 / 京东方A` 写先进封装/TGV 旁证位。
6. 材料和平台样本只做旁证，不抢设备主结论位。
7. 四批样本第一轮公告、财报、IR 回指完成后，后续维护优先走 `半导体设备全景组滚动验证清单.md` 的中报 / 三季报连续验证表，而不是继续横向扩名字。
8. 阶段切换必须同时过 `订单 / 客户 / 交付` 与 `收入 / 利润 / OCF` 两段验证，不能用单点公告直接抬升组内定位。
9. 每次财报后都要补 `资本周期审计`，固定回看 `OCF / 应收 / 存货 / 预付款 / CapEx / 在建工程 / 研发费用率`。

## 六、下一步维护原则

1. 先更新滚动清单，再更新旁证证据表，最后再更新组主文。
2. 新增上游样本先按 `晶圆制造 / 先进封装 / 高速互联 / AIDC` 路由，再判断其是原材料、关键零部件、制程系统还是资源供给。
3. 再判断它是核心交叉验证、供给安全 / 长周期卡点、弹性发现还是覆盖 / 反证样本。
4. 不把旁证写成主线。
5. 不把题材写成兑现。
6. 不把单点公告写成终局结论。
7. 不跳过连续验证阶段，单次回指完成不等于兑现链条已经闭环。
8. 如果阶段口径、顺序或标签变化，同步更新索引页与总映射索引，避免新旧口径分裂。
