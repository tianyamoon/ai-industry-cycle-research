# 研究文档头部字段模板_第一版

> 形成日期：2026-06-29  
> 目标：给单卡、组比较、总表、阶段收口建立统一头部字段，降低接管成本，提升机读性。

---

## 一、统一字段枚举

建议后续新文档或大幅更新文档时，优先补以下字段：

1. `文档类型`
2. `研究对象`
3. `所属模块`
4. `所属层级`
5. `周期角色`
6. `第一变量`
7. `验证变量`
8. `证据完成度`
9. `当前状态`
10. `上收状态`
11. `权威级别`
12. `关联文件`

---

## 二、推荐枚举值

### 1. 文档类型

1. `单卡`
2. `组比较`
3. `总表`
4. `阶段收口`
5. `总论主文`
6. `专题观察`
7. `滚动验证清单`

### 2. 所属层级

1. `过程层`
2. `桥接层`
3. `结果层`
4. `交接层`

### 3. 证据完成度

1. `价格完成`
2. `财务部分完成`
3. `资金待核`
4. `反证已补`
5. `待补较多`

### 4. 当前状态

1. `待建`
2. `已建`
3. `已比较`
4. `已上收`
5. `观察池`
6. `历史归档`

### 5. 上收状态

1. `未进入上层总结`
2. `已进入模块索引`
3. `已进入总论`

### 6. 权威级别

1. `状态源`
2. `模块主入口`
3. `权威主文`
4. `执行层文档`
5. `历史归档`

---

## 三、单卡模板

```md
> 文档类型：单卡
> 研究对象：公司名
> 所属模块：02_桥接层/03_AI标杆股研究卡
> 所属层级：桥接层
> 周期角色：核心资产 / 平台承接 / 资源运营 / 基础设施协同 / 应用成熟 / 阴影反证
> 第一变量：一句话
> 验证变量：订单 / ASP / 毛利率 / OCF / CapEx / 客户结构 / 资金结构
> 证据完成度：价格完成 / 财务部分完成 / 资金待核
> 当前状态：已建
> 上收状态：已进入模块索引
> 权威级别：执行层文档
> 关联文件：组比较、图谱、总论、主表
```

---

## 四、组比较模板

```md
> 文档类型：组比较
> 研究对象：组名
> 所属模块：02_桥接层/03_AI标杆股研究卡
> 所属层级：桥接层
> 周期角色：写组在图谱中的定位
> 第一变量：一句话
> 验证变量：订单 / ASP / 毛利率 / OCF / CapEx / 客户结构 / 融资结构
> 证据完成度：价格完成 / 财务部分完成 / 资金待核 / 反证已补
> 当前状态：已比较
> 上收状态：已进入模块索引 / 已进入总论
> 权威级别：执行层文档
> 关联文件：组内单卡、图谱、总论、交接包
```

---

## 五、总表 / 阶段收口模板

```md
> 文档类型：总表 / 阶段收口
> 研究对象：总表名或阶段名
> 所属模块：01_过程层/04_新能源数据闭环补证 或 03_结果层/01_总表整合与AI映射
> 所属层级：过程层 / 结果层
> 周期角色：写该表解决哪类问题
> 第一变量：一句话
> 验证变量：价格 / 财务 / 资金 / 反证 / 映射
> 证据完成度：价格完成 / 财务部分完成 / 资金待核 / 反证已补
> 当前状态：已上收
> 上收状态：已进入总论
> 权威级别：权威主文
> 关联文件：主索引、状态面板、模块入口、交接包
```

---

## 六、使用规则

1. 不要求一次性把所有旧文档重做。
2. 优先在新增文档、滚动更新的主入口文档和未来重写文档中采用。
3. 若字段信息不完整，可以先显式写 `待补`，不要省略掉字段位。
4. 若文档已进入上层总结，`上收状态` 必须明确写出。
