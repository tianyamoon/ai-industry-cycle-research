# 财经资料源与搜索工具集_第一版

> 形成日期：2026-07-05  
> 所属层级：`00_总控台 / 过程层证据采集工具`  
> 目标：把财经资料源按变量类型和证据强度固定下来，后续研究先按变量选源，再搜索和落表，避免被搜索引擎随机带到低质量转载源。

---

## 一、总原则

一句话规则：

`先确定变量 -> 再选资料源 -> 再搜索原文 -> 最后才组织判断`

后续所有行业、公司、行情和政策研究，默认按下面顺序取证：

1. `S级：原始披露 / 法定披露 / 官方统计`
   - 公司公告、定期报告、交易所披露、监管文件、政府统计、公司 IR 原文。
   - 可以直接支撑事实层，但仍需保留日期、单位、链接和口径。
2. `A级：专业数据库 / 行业协会 / 价格机构`
   - Wind、Choice、iFinD、Bloomberg、FactSet、SMM、Mysteel、百川盈孚、PVInfoLink、SEMI、WSTS、SIA、IDC、Gartner、Omdia 等。
   - 可支撑变量层和验证层；若无可公开链接，必须标注为 `终端口径 / 待导出留痕`。
3. `B级：主流财经媒体 / 通讯社 / 专业行业媒体`
   - Reuters、Bloomberg、FT、Nikkei、财联社、证券时报、中国证券报、上海证券报、财新、21世纪经济报道、DIGITIMES 等。
   - 主要用于还原当时市场叙事、事件线索和市场理解；不能单独替代公告、财报和数据库。
4. `C级：门户 / 聚合 / 社区 / 转载`
   - 新浪财经、腾讯财经、网易财经、搜狐财经、雪球、股吧、公众号转载、未标明来源的研报摘录。
   - 只能当线索，不得直接支撑主结论。若内容有价值，必须回溯到公告、交易所、公司官网、行业协会或原媒体。
5. `D级：不可用`
   - 无来源截图、群聊传闻、二手剪贴、无法确认日期和出处的表格。
   - 不进入研究正文，只能写入 `待核`。

## 三、搜索路由模板

### 1. K线和行情数据

优先调用：

```text
本地股票数据 MCP：get_index_kline / 个股K线能力 / get_stock_detail
```

使用顺序：

1. 个股或指数 K 线、成交额、换手率、区间涨跌幅，先走本地 MCP；
2. MCP 结果完整时，可作为第一版行情证据；
3. 需要形成正式结论时，再用 Wind / Choice / iFinD / 东方财富 / 交易所行情做口径复核；
4. 若 MCP 返回 `ok:true` 但数据为空、字段缺失、日期不连续或复权口径不清，视为 `待核`，不得直接落成强结论；
5. 指数清单、主题详情、聚合板块概览等接口若出现空成功或 schema 漂移，必须改用单源验证或人工确认。

落表字段：

```text
标的 | 代码 | 区间 | 复权口径 | 开高低收 | 成交额 | 换手率 | 区间涨跌幅 | 数据工具 | 交叉来源 | 待核说明
```

### 2. 东方财富、同花顺、通达信

优先用途：

```text
东方财富：数据中心 / 行情中心 / 期货频道 / 研报中心 / 估值分析 / 龙虎榜 / 资金流 / 公告镜像
同花顺：财经资讯 / 问财 / 板块概念 / 涨跌停 / 资金流 / iFinD
通达信：本地行情 / 技术指标 / 公式系统 / 板块指数 / 条件选股 / 历史复盘
```

使用顺序：

1. 盘面复盘、板块强弱、涨跌停、龙虎榜、资金流和技术指标，优先看 `MCP + 东方财富 / 同花顺 / 通达信`；
2. 通达信用于图形、指标、条件选股和板块联动，不用于证明财务事实；
3. 东方财富和同花顺的公告、研报、资讯可以提供入口，但正式引用要追到交易所、巨潮、公司公告或原始研报；
4. 资金流、主力净流入、概念归属、热度排名属于交易线索，不能直接写成产业验证；
5. 估值、总市值、换手率等行情字段可用作第一版，但强结论要交叉 Wind / Choice / iFinD / 交易所或 MCP。

落表字段：

```text
工具 | 字段 | 标的/板块 | 日期 | 数值/状态 | 用途 | 是否可直接引用 | 需要追溯的原始源
```

品种 期货 文华财经

```

使用顺序：

1. 结算价、成交量、持仓量、仓单、保证金、交割规则，先看交易所；
2. 夜盘异动、跨品种联动、期现价差线索，可用文华、东方财富期货、金十、99期货、Mysteel；
3. 若用于产业价格验证，必须区分 `期货价格`、`现货价格`、`指数价格`、`连续合约` 和 `主力合约`；
4. 不能用期货短线涨跌直接替代现货供需判断。

### 4. 美股资讯和行情

```

使用顺序：

1. 美股行情、盘前盘后、财报日历、新闻反应，可先用 Yahoo Finance / MarketWatch / Nasdaq / CNBC；
2. 技术图形和跨市场对照，可用 TradingView / Finviz；
3. 市场观点和分歧，可参考 Seeking Alpha、MarketWatch、CNBC，但只能当观点层；
4. 公司收入、CapEx、毛利率、订单、backlog、风险因素，仍必须回到 SEC / 公司 IR。

### 4. 盯站动作

1. 每周至少做一次全量扫站：海外投行、全球资管 / 宏观、国内头部卖方三张表都要扫。
2. 每个交易日若出现 `FOMC / CPI / 非农 / 财报季 / AI大厂CapEx / 商品价格大波动 / 地缘冲击 / 半导体限制政策`，当天加扫相关机构。
3. 对 AI、半导体、能源、电力、金属、信用、利率这几类主线，优先记录 `最新报告标题 + 日期 + 机构 + 核心变量 + 与当前仓库结论的关系`。
4. `商业研报` 默认归入每日跟踪，交易日优先扫 `东方财富行业研报`，只把它当作 `机构观点 / 行业线索 / 变量提示` 入口，不直接当成事实层证据。
5. 只看到摘要时，落为 `摘要口径`；看到客户门户或终端摘录时，落为 `终端口径 / 待核全文`；拿到公开 PDF 或官网正文时，才落为 `公开原文`。
6. 同一主题若只有单家机构观点，只能进入观察层；形成正式判断前必须补到 `2家以上顶级机构 + 1条原始事实源`。

### 5. 研报进入研究结论的门槛

1. 研报只能先进入 `预期层 / 方法论层 / 市场叙事层`。
2. 研报里的预测要拆成假设，不能写成事实。
3. 研报里的公司财务、价格、订单、CapEx，要回到公告、财报、价格库、公司 IR 或统计源。
4. 多家机构观点分歧必须保留，不能只摘最支持自己结论的一家。
5. 对 AI、半导体、能源、大宗商品等高分歧方向，至少保留：
   - 2 家全球投行；
   - 1 家资管 / 宏观机构；
   - 1 家国内头部卖方；
   - 1 条原始事实源。

---

## 六、市场信息平台处理规则

### 1. 东方财富怎么用

东方财富不是低质量源，应该分三层：

1. `数据中心 / 行情中心 / 期货频道 / 估值分析 / 龙虎榜`：适合盘面复盘和第一版结构化数据；
2. `Choice / 研报中心 / 公告大全`：适合做专业入口和索引，但要注意授权、口径和原始出处；
3. `股吧 / 评论 / 资讯转载`：只能做情绪和线索。

使用边界：

1. 行情字段可以做第一版，但强结论要复核；
2. 公告镜像必须追巨潮或交易所原文；
3. 研报摘要必须追完整研报或标为 `摘要口径`。

### 2. 同花顺怎么用

同花顺也要分层：

1. `iFinD`：专业数据库层，可与 Wind / Choice 并列；
2. `问财 / 概念板块 / 涨跌停 / 资金流`：盘面工具层；
3. `财经资讯 / 社区内容`：线索和叙事层。

使用边界：

1. 问财适合快速发现样本和字段，不直接替代原始披露；
2. 概念归属和资金流是交易线索，不是产业事实；
3. iFinD 导出数据要标明字段、口径和导出时间。

### 3. 通达信怎么用

通达信的定位是 `本地行情 + 技术分析 + 板块联动 + 条件选股`。

适合：

1. K线图形复盘；
2. 板块指数和概念联动；
3. 技术指标、公式、条件选股；
4. 历史行情文件与本地回放。

不适合：

1. 证明公司收入、利润、订单、CapEx；
2. 替代公告、财报或交易所披露；
3. 用软件自带概念标签直接写产业结论。

### 4. 期货信息网站怎么用

期货信息源按三层使用：

1. `交易所`：结算价、成交、持仓、仓单、合约规则，证据等级 S；
2. `文华 / 东方财富期货 / 博易大师`：盘面、技术图形、跨品种盯盘，证据等级 A/B；
3. `金十 / 99期货 / 汇通 / 期货媒体`：夜盘异动、快讯、评论和交易叙事，证据等级 B/C。

若要支撑产业结论，必须继续补：

1. 现货价格；
2. 库存；
3. 开工率；
4. 产量 / 进口 / 出口；
5. 公司财报或订单。

### 5. 美股资讯网站怎么用

美股资讯源按三层使用：

1. `SEC / 公司 IR / 交易所`：事实层；
2. `Yahoo Finance / MarketWatch / Nasdaq / CNBC / TradingView / Finviz`：行情、日历、筛选和叙事层；
3. `Seeking Alpha / 社区评论 / 博客`：观点和分歧层。

使用边界：

1. 财报日历和盘后反应可以用资讯站；
2. 财报数字、CapEx、订单和风险因素必须回 SEC / 公司 IR；
3. 分析师观点和社区文章只能进入 `市场预期 / 分歧`，不能直接写成事实。

## 七、低质量来源处理规则

### 1. 新浪等门户怎么用

新浪、腾讯、网易、搜狐这类入口只能做三件事：

1. 找关键词；
2. 找原始媒体或公告名称；
3. 观察市场叙事扩散。

不得做三件事：

1. 不得把门户转载当作公告原文；
2. 不得用门户标题直接写产业结论；
3. 不得在没有原始链接时填精确数据。

### 2. 公众号和社区怎么用

公众号、雪球、股吧适合发现市场正在讨论什么，但不适合证明产业发生了什么。

使用规则：

1. 只记录为 `市场叙事线索`；
2. 不能写成 `事实`；
3. 若其中出现数据，必须回溯到公告、数据库、行业协会或原媒体。

## 八、落表标准字段

所有后续补证表、研究卡和总表，建议统一保留下面字段：

| 字段 | 说明 |
|---|---|
| 变量 | 需求、供给、价格、库存、CapEx、财务、估值、资金、政策、技术路线 |
| 日期 / 期间 | 精确到交易日、月份、季度或年度 |
| 数值 | 不确定时不填，写 `待核` |
| 单位 | 亿元、万吨、GW、GWh、元/吨、台、片、美元、% 等 |
| 口径 | 同比、环比、TTM、MRQ、前复权、未复权、合并口径、单体口径 |
| 来源名称 | 公告名、数据库名、协会名、媒体名 |
| 来源链接 | 能放链接必须放链接；终端数据写导出文件名或截图路径 |
| 证据等级 | S / A / B / C / D |
| 是否原始来源 | 是 / 否 |
| 是否交叉验证 | 是 / 否 / 待核 |
| 当前用途 | 事实层、变量层、传导层、验证层、映射层 |
| 待补说明 | 缺哪个口径、缺哪个期间、缺哪个交叉来源 |

## 九、执行纪律

1. 搜索前先写变量，不先搜结论。
2. K线、成交、指数行情先调本地 MCP，再做口径复核。
3. 盘面、板块、资金流和技术分析，应优先使用东方财富、同花顺、通达信等市场信息工具，但必须标明用途是 `盘面层 / 交易层`。
4. 期货结算价、持仓、仓单和合约规则先看交易所；期货资讯网站只做行情线索和交易叙事。
5. 美股行情和财报日历可用 Yahoo Finance、MarketWatch、Nasdaq、CNBC 等；公司事实仍回 SEC / 公司 IR。
6. 顶级金融机构研报必须作为重点跟踪对象；每周扫公开入口，重大事件后加扫。
7. 研报进入结论前必须拆成 `事实 / 假设 / 估值框架 / 交易观点 / 风险`。
8. 公司事实先找公告和财报，不先找门户新闻。
9. 政策先找部门原文，不先找解读稿。
10. 价格先找连续数据库，不拼接没有口径的新闻数字。
11. 行业数据先找协会、统计部门、专业机构，不用社区截图。
12. 新闻只解决“当时市场在交易什么”，不直接解决“产业是否验证”。
13. 聚合站只做线索，除非它提供的是可复核的行情字段且已交叉验证。
14. 多来源冲突时，保留差异，采用更保守表述。
15. 缺资料时写 `待补 / 待核 / 待确认`，不把空白补成故事。
16. 每次形成强结论，至少能回到一条 `S级` 或 `A级` 来源链。
