# 文档发布与状态源规则

> 更新日期：2026-07-07

## 唯一状态源

- 研究状态：`00_总控台/项目状态面板.md`
- 整理状态：`00_总控台/项目治理看板.md`

二者分开维护，不混写。

## 顶层接口

- `README.md`：总导航与阅读顺序，不承载长期状态
- `项目状态面板.md`：研究进度、主线、优先级
- `项目治理看板.md`：整理进度、allowlist、完成标准
- `90_历史归档/`：历史输出唯一落点
- `90_执行留痕/`：执行过程唯一落点
- `tmp/`：临时、缓存、实验唯一隔离区

## 活文档命名规则

- 活文档使用稳定文件名，直接在原文件上迭代
- dated 输出只保留日期，不保留 `第一版/第二版/第三版`
- 若日期本身就是产物语义的一部分，例如周评分、周审计、阶段补证主文、纪要、线索池，可作为稳定 dated 主文留在活跃目录
- 历史演进默认交给 Git 和 `90_*` 目录
- 禁止在活跃目录并行保留多个版号文件
- 同主题但不同角色的当前文件允许并存，例如 `主卡` 与 `回填补证卡`

## 目录角色规则

- `00_总控台`：方法论、规则、状态源、总映射索引
- `01_过程层`：证据生产、历史复盘、硬数据补证、图表素材
- `02_桥接层`：研究卡、样本池、组比较、滚动验证
- `03_结果层`：正式总结、总表整合、正式写作
- `99_交接包`：接管入口与历史交接

## 根目录规则

- 根目录只保留固定目录和必要仓库文件
- 不再在根目录放置 `.tmp*`、`.tmp_*`、`isc_*`、散落 `tmp_*` 实验目录
- 根目录产生的缓存进入 `tmp/root-cache/`
- 根目录产生的 scratch 目录进入 `tmp/root-scratch/`
- 根目录实验文件进入 `tmp/experiments/`

## README 规则

每个重点模块 README 只回答两件事：

1. 现在先看什么
2. 历史在哪里

README 不再承担长期状态源角色。

## 历史与留痕规则

- 正式历史输出进入 `90_历史归档/`
- 执行过程、批处理计划、阶段试跑进入 `90_执行留痕/`
- 会话接管材料保留在 `99_交接包/`，但当前入口严格控制在 `1-3` 个

## 最小回写原则

- 只补证据：优先回写过程层主表或索引
- 新增样本组：回写桥接层主索引与总映射索引
- 形成正式结论：回写结果层主文，并同步必要索引
- 发生治理动作：回写 `项目治理看板.md`，而不是挤进研究状态面板
