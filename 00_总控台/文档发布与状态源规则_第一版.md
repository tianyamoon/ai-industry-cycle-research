# 文档发布与状态源规则_第一版

> 形成日期：2026-06-29  
> 目标：把仓库文档固定成 `状态源 + 过程层 + 桥接层 + 结果层 + 交接层` 的清晰结构，减少重复入口和多份真相。

---

## 一、唯一状态源

本项目当前固定以：

`00_总控台/项目状态面板.md`

作为唯一权威状态源。

凡涉及以下问题，均以该文件为准：

1. 当前做到哪里
2. 当前主线是什么
3. 当前优先级是什么
4. 哪些缺口阻塞，哪些不阻塞
5. 哪些判断已经可以进入结果层

## 三、文档分层规则

### 1. 状态源层

只有一份：

1. `00_总控台/项目状态面板.md`

### 2. 过程层

用于生产研究证据，而不是发布最终结论。

典型文档：

1. 硬数据主表
2. 时间轴索引
3. 补证计划
4. 历史复盘资料
5. 图表素材

对应目录：

1. `01_过程层/01_新能源十年周期母体/`
2. `01_过程层/02_新能源周期复盘/`
3. `01_过程层/03_历史新闻流与政策流/`
4. `01_过程层/04_新能源数据闭环补证/`
5. `01_过程层/05_图表与配图/`

### 3. 桥接层

用于把过程层压成标准研究骨架，先服务研究，再决定是否升级。

典型文档：

1. 图谱主文
2. 单卡
3. 组比较
4. 滚动验证清单
5. 标准行业研究卡

对应目录：

1. `02_桥接层/01_新能源标杆股研究卡/`
2. `02_桥接层/02_AI产业链研究卡/`
3. `02_桥接层/03_AI标杆股研究卡/`

### 4. 结果层

用于承接已经完成证据链、变量链、验证链的正式输出。

典型文档：

1. 正式总结
2. 总映射正式版
3. 阶段总结
4. 总表整合
5. 正式正文

对应目录：

1. `03_结果层/01_总表整合与AI映射/`
2. `03_结果层/02_正式写作区/`

### 5. 交接层

用于保留历史轨迹与接管信息，但不与正式入口争夺位置。

对应目录：

1. `99_交接包/01_当前接管入口/`
2. `99_交接包/90_历史归档/`

## 六、最小回写集合

### 情况1：只补局部证据

例如：

1. 补价格月度数据
2. 补季度财务
3. 补单条公告或新闻

最小回写集合：

1. 对应过程层主表或索引
2. 若结论边界变化，再回写桥接层索引或结果层主文

默认不要求更新：

1. `README.md`
2. `99_交接包/README.md`
3. `整体工作与目标路线图.md`

### 情况2：新增一组样本或一条新子链

最小回写集合：

1. 对应桥接层组主文 / 单卡 / 组比较
2. `02_桥接层/03_AI标杆股研究卡/AI标杆股研究卡_进度索引.md`
3. `00_总控台/坐标_样本_证据_结论总映射索引_第一版.md`
4. 如影响主线优先级，再更新 `项目状态面板.md`

### 情况3：改变项目主线、阶段判断或优先级

最小回写集合：

1. `00_总控台/项目状态面板.md`
2. `00_总控台/整体工作与目标路线图.md`
3. 对应模块入口文档
4. 如接管路径明显变化，再更新 `README.md`

### 情况4：形成阶段性上收或正式结项

最小回写集合：

1. 结果层权威主文
2. `项目状态面板.md`
3. 对应模块索引
4. `99_交接包/01_当前接管入口/` 新增交接包

---

## 七、何时必须写交接包

满足以下任一条件时，应考虑新增 `99_交接包/01_当前接管入口/`：

1. 形成新的阶段收口
2. 完成一组样本并影响后续优先级
3. 完成一份可直接接管的总论或专题入口
4. 主线顺序发生变化
5. 下一会话若不读交接包就容易误判当前状态

---

## 八、何时不要平行新建

默认优先更新已有：

1. 过程层主表
2. 桥接层主索引
3. 结果层主文
4. 总控台总映射索引

只有在以下情况才新增文档：

1. 新对象确实是新样本 / 新组别 / 新专题
2. 旧文档职责已满，继续写会破坏清晰分层
3. 新文档能明确归入既有层级，而不是新增平行入口

---

## 九、发布纪律

后续所有结构更新，默认遵循四条纪律：

1. 先更新状态源，再更新导航入口。
2. 先更新过程层和桥接层，再决定是否上收为结果层。
3. 先回写索引和主表，再补旁支说明。
4. 先减少重复口径，再增加新内容。
