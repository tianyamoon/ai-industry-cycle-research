# 研究框架

## 项目名称

AI产业周期研究

## 当前总框架

本仓库当前固定按五层理解：

1. `00_总控台`
   方法论、状态源、模板、规则、总映射索引。
2. `01_过程层`
   研究方法论落地、历史母体、时间轴资料、硬证据补证、图表素材。
3. `02_桥接层`
   标准行业研究卡、产业链图谱、样本验证卡、组比较、滚动验证清单。
4. `03_结果层`
   正式总结、总表整合、阶段上收、正式写作区。
5. `99_交接包`
   阶段留痕、当前接管入口、历史归档。

其中真正的研究主链只有三层：

`过程层 -> 桥接层 -> 结果层`

---

## 一、三层各自解决什么问题

### 1. 过程层

解决：

`研究怎么做，证据怎么生产，历史怎么还原`

当前承载内容：

1. 新能源十年周期母体
2. 新能源周期复盘
3. 历史新闻流与政策流
4. 新能源数据闭环补证
5. 图表与配图

默认输出物：

1. 变量表
2. 钱包份额表
3. 证据索引
4. 硬数据主表
5. 时间轴资料
6. 补证计划

### 2. 桥接层

解决：

`如何把过程层证据压成可比较、可升级的标准研究卡`

当前承载内容：

1. 新能源标杆股研究卡
2. AI产业链研究卡
3. AI标杆股研究卡

默认输出物：

1. 图谱坐标
2. 单卡
3. 组比较
4. 滚动验证清单
5. 阶段性标准研究骨架

### 3. 结果层

解决：

`如何把桥接层样本和过程层证据，组织成正式可引用输出`

当前承载内容：

1. 总表整合与AI映射
2. 正式写作区

默认输出物：

1. 新能源正式总结
2. AI总映射正式版
3. 阶段总结
4. 正式正文

---

## 二、当前主线顺序

当前固定顺序不是“想到哪写到哪”，而是：

`过程层硬证据 -> 结果层新能源正式总结 -> 桥接层AI图谱 -> 桥接层AI样本验证 -> 结果层AI总映射`

这条链条背后的含义是：

1. 先把新能源母体证据压实；
2. 再把新能源正式总结作为方法母体固定下来；
3. 再用桥接层AI图谱定义 AI 的统一坐标；
4. 再由 AI 样本验证层按坐标补组、补卡、补反证；
5. 最后才上收为结果层正式总论。

## 三、当前模块定位

### 00_总控台

职责：

1. 方法论校准
2. 项目状态发布
3. 文档结构规则
4. 模板统一
5. 总映射索引

### 01_过程层

职责：

1. 生产研究证据
2. 还原历史信息环境
3. 建立新能源母体硬数据闭环
4. 提供图表和原始资料基础

### 02_桥接层

职责：

1. 把过程层证据压缩成标准研究卡
2. 统一 AI 图谱坐标
3. 汇总样本验证与反证
4. 为结果层正式总论做结构化上游供给

### 03_结果层

职责：

1. 形成正式总结
2. 形成总表整合
3. 形成阶段上收
4. 组织正式正文

### 99_交接包

职责：

1. 保留阶段轨迹
2. 提供少量当前接管入口
3. 记录未决问题与下一步建议

## 五、执行纪律

1. 过程层不直接伪装成正式结论。
2. 桥接层先做研究骨架，再决定是否升级成正式成稿。
3. 结果层只承接已经有证据链、变量链、验证链的内容。
4. 交接包保留历史轨迹，但不与状态源争夺主入口。
5. 若一个判断说不清 `事实 -> 变量 -> 传导 -> 验证 -> 映射`，就不要急着进入结果层。

---

## 六、当前最需要坚持的边界

1. 不再用旧编号理解优先级，而用层级理解职责。
2. 不把桥接层的观察池、专题观察直接写成结果层正式结论。
3. 不把过程层补证文档当成最终可读成稿。
4. 不在结果层重复堆积原始证据。
5. 不平行新建重复入口，优先回写现有主索引、主表与主文。
