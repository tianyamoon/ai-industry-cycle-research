# 顶级投资机构产业逻辑研究打分框架

> 形成日期：2026-07-02
> 目标：基于国外 `10` 篇、国内 `10` 篇公开研究样本，提炼一套可给“当下产业逻辑研究”打分、找缺口、排补证优先级的统一框架。
> 定位：这是 `研究质量审计工具`，不是替代研究本身的结论生成器。

---

## 一、样本使用说明

1. 国外样本优先采用机构官网原文或官网研究入口。
2. 国内样本因公开可得性差异较大，区分为：
   - `官网原文`
   - `官网报道 / 官网研究页`
   - `研报平台摘要`
3. 对于仅能拿到摘要的平台样本，只用于提炼方法结构，不用于逐字引用强结论。
4. 后续若拿到更完整的 PDF 原文，应优先回写本文件。

## 三、20 篇报告最稳定的共性

这些报告风格差异很大，但真正稳定重复出现的，不是语言，而是研究顺序。

### 1. 先讲“现在站在哪”，再讲“未来会怎样”

顶级报告几乎不会跳过阶段判断。它们先回答：

1. 当前更像导入、验证、扩散、拥挤、兑现还是退潮。
2. 市场是在交易改善，还是在透支改善。
3. 当前的主要矛盾是需求、供给、融资、估值，还是情绪。

### 2. 先缩变量，再扩结论

优秀报告并不追求变量越多越好，而是优先锁定：

1. `主导变量`
2. `验证变量`
3. `滞后变量`
4. `证伪变量`

如果一篇研究没法明确这四类变量，往往只是叙事堆积。

### 3. 一定把大主题拆成链条和层次

无论是 AI、能源转型还是宏观再通胀，顶级机构都会继续追问：

1. 钱到底花在哪一层。
2. 谁吃总钱包。
3. 谁吃增量钱包。
4. 谁最后留下利润池。
5. 谁只是放量但赚不到钱。

### 4. 供给、CapEx 和资本成本几乎从不缺席

真正成熟的研究不会只看需求。

它们一定会问：

1. 高利润是否正在吸引新供给。
2. 新供给何时释放。
3. 融资条件是否仍允许资本继续涌入。
4. 资本开支会不会先推高景气、后压垮回报率。

### 5. 不把增长自动等同于价值创造

国外样本尤其强调：

1. 增长只有在 `增量回报率 > 资本成本` 时才创造价值。
2. 市占率提升如果靠牺牲回报率换来，不一定值得更高估值。
3. 现金流、ROIC、资本配置，比单纯收入更能区分穿越者。

### 6. 一定审计市场已经隐含了什么

顶级机构会把“研究对不对”和“投资值不值”分开。

常见动作是：

1. 反推当前价格已隐含的远期假设。
2. 检查估值是否已经提前透支产业改善。
3. 单独讨论拥挤度、风格偏好和交易结构。

### 7. 一定保留反证和情景树

优质研究不追求单一路径的流畅感，而追求：

1. 乐观、基准、悲观三种场景。
2. 哪个变量一旦失真，结论会被推翻。
3. 哪些风险已经在价格里，哪些风险还没在价格里。

### 8. 历史参照不是拿来套图形，而是拿来套机制

共性非常明显：

1. 历史比较用于找 `顺序` 与 `约束`
2. 不用于机械复刻时点和涨幅
3. 更重视资本周期、供需失衡、估值泡沫这些机制映射

### 9. 写法都偏“可监控”

好的报告最后通常会落到：

1. 下一个验证节点是什么
2. 应该盯哪些月度 / 季度指标
3. 哪个公告、数据、价格、财报最关键

也就是说，研究不是写完就结束，而是默认进入滚动验证。

### 10. 真正的差异不在文风，而在“结论前是否经过了审计链”

这 20 篇报告最重要的共性可以压缩成一句话：

`阶段判断 -> 变量拆解 -> 传导链 -> 资本周期 -> 公司经济性 -> 估值预期 -> 风险反证 -> 监控节点`

谁缺这条链，谁的研究更容易变成“看上去很完整，实际上难以验证”的结论。

## 四、提炼成可打分的统一框架

### 总分：100 分

| 维度 | 权重 | 核心问题 |
|---|---:|---|
| 1. 问题定义与边界 | 8 | 研究到底在回答什么，不回答什么 |
| 2. 结构力量与宏观背景 | 7 | 这是结构性趋势、流动性行情还是普通景气波动 |
| 3. 核心变量拆解 | 15 | 需求、供给、成本、价格、CapEx、库存、资金谁主导 |
| 4. 传导链完整性 | 12 | 变量如何传导到景气、利润、估值和股价 |
| 5. 供给 / CapEx / 资本周期审计 | 12 | 扩产、融资和回报反噬有没有被审计 |
| 6. 钱包份额 / 利润池 / 公司经济性 | 12 | 谁吃收入，谁留利润，谁能转成现金流和 ROIC |
| 7. 阶段定位与历史映射 | 10 | 当前处在哪一段，历史类比是机制类比还是图形类比 |
| 8. 估值 / 预期 / 资金拥挤 | 12 | 市场已隐含什么，结论对但投资也可能错在哪里 |
| 9. 证据质量与可证伪性 | 7 | 日期、单位、来源、口径都清楚吗 |
| 10. 风险、监控点与后续验证计划 | 5 | 研究能不能进入滚动维护 |

### 分数锚点

| 分数段 | 判断 |
|---|---|
| `90-100` | 顶级研究框架，可直接进入正式总论或高质量投资讨论 |
| `80-89` | 结构扎实，少数证据或估值环节仍需补强 |
| `70-79` | 可用，但更多是“研究骨架”，还不够支撑重结论 |
| `60-69` | 叙事大于审计，容易把主题热度误写成产业逻辑 |
| `<60` | 不宜上收为正式判断，必须先补变量、证据或估值链 |

## 五、每个维度怎么打

### 1. 问题定义与边界｜8 分

高分特征：

1. 明确研究对象、时间框架、产业链边界和不回答的问题。
2. 能区分行业判断、公司判断和市场判断。

低分症状：

1. 主题很大，但没有边界。
2. 同时混写产业、个股、政策和股价，不分层。

### 2. 结构力量与宏观背景｜7 分

高分特征：

1. 能分清结构性趋势和短期资金推动。
2. 能交代利率、信用、流动性、政策对结论的放大或压制作用。

低分症状：

1. 只讲产业，不讲资本成本和融资环境。
2. 默认任何高增长都可以线性持续。

### 3. 核心变量拆解｜15 分

高分特征：

1. 清楚列出主导变量、验证变量、滞后变量、证伪变量。
2. 能说明谁是噪音，谁决定结论。

低分症状：

1. 数据很多，但主变量不清楚。
2. 研究靠“案例堆叠”，不是变量组织。

### 4. 传导链完整性｜12 分

高分特征：

1. 能把研究还原为 `事实 -> 变量 -> 产业景气 -> 公司报表 -> 估值 -> 股价`。
2. 能解释哪个环节先反应，哪个环节后验证。

低分症状：

1. 直接从新闻跳到股价判断。
2. 没有说明产业变化如何变成利润变化。

### 5. 供给 / CapEx / 资本周期审计｜12 分

高分特征：

1. 能审计扩产、融资、供给释放、价格战和出清风险。
2. 对重资产行业会单独审计折旧、回款、现金流和负债结构。

低分症状：

1. 只看订单和需求，不看新增产能。
2. 只看收入高增，不看资本开支反噬。

### 6. 钱包份额 / 利润池 / 公司经济性｜12 分

高分特征：

1. 能分清收入承接者和利润留存者。
2. 能讨论 ROIC、毛利率、自由现金流、资本配置。

低分症状：

1. 把所有链条都写成同步受益。
2. 不区分放量受益和高回报受益。

### 7. 阶段定位与历史映射｜10 分

高分特征：

1. 明确当前更接近导入、验证、扩散、拥挤、兑现还是退潮。
2. 历史比较用于解释顺序和机制，不乱套时间表。

低分症状：

1. 只有类比，没有阶段。
2. 只拿 K 线形态说故事。

### 8. 估值 / 预期 / 资金拥挤｜12 分

高分特征：

1. 会问价格里已经隐含了什么。
2. 能解释为什么“逻辑对，但股票未必值得买”。

低分症状：

1. 完全不讨论估值和持仓拥挤。
2. 把产业向上直接等同于投资回报向上。

### 9. 证据质量与可证伪性｜7 分

高分特征：

1. 日期、单位、来源、口径都清楚。
2. 明确写出 `待补`、`待核`、`证伪条件`。

低分症状：

1. 模糊词很多，硬证据很少。
2. 没有来源，也没有时间锚点。

### 10. 风险、监控点与后续验证计划｜5 分

高分特征：

1. 有明确跟踪清单和下一个验证节点。
2. 能说清哪些数据出来后，分数要上调或下调。

低分症状：

1. 只有结论，没有维护机制。
2. 研究无法进入周更、季更或财报后回写。

## 六、硬性降分与封顶规则

出现下列问题时，即使文字写得流畅，总分也应封顶：

1. `没有阶段判断`
   - 总分封顶 `75`
2. `没有估值 / 预期 / 资金拥挤讨论`
   - 总分封顶 `72`
3. `重资产或扩产行业没有 CapEx / 供给审计`
   - 总分封顶 `68`
4. `没有明确日期、来源、单位`
   - 总分封顶 `60`
5. `没有反证条件或主要风险`
   - 总分封顶 `78`
6. `只写龙头逻辑，不写失败样本或阴影路径`
   - 总分额外扣 `5-10`

## 七、常见不足与修复顺序

### 1. 常见不足

1. 只讲需求，不讲供给。
2. 只讲产业，不讲股价和估值。
3. 只讲龙头，不讲阴影资产。
4. 只讲景气，不讲现金流和 ROIC。
5. 只有共识，不写反证。
6. 只有结论，没有下一步监控表。

### 2. 默认修复顺序

1. 先补 `阶段判断`
2. 再补 `主变量表`
3. 再补 `传导链`
4. 再补 `CapEx / 供给审计`
5. 再补 `估值 / 资金 / 拥挤`
6. 最后补 `反证条件与监控点`

## 八、标准输出模板

给一篇当下产业逻辑研究打分时，默认输出为：

1. `一句话总评`
   - 当前结论是否成立，最薄弱的一环是什么
2. `总分 + 等级`
3. `分维度评分表`
4. `前三个优点`
5. `前三个缺口`
6. `硬性封顶项`
7. `最该优先补的证据`
8. `哪个变量最可能推翻当前判断`
9. `如果补齐什么，分数可再上调多少`

## 九、在本仓库里的推荐用法

### 1. 过程层

优先用来审计：

1. 变量表是否完整
2. 证据链是否能支撑阶段判断
3. 是否误把“新闻热度”当成“产业事实”

### 2. 桥接层

优先用来审计：

1. 每张研究卡是不是有统一坐标
2. 有没有把“钱包份额”和“利润池”分开
3. 有没有从观察池误升格成正式样本

### 3. 结果层

优先用来审计：

1. 是否过度上收
2. 是否把待补证据说成已验证
3. 是否遗漏估值、拥挤、资本周期反噬

---

## 十、当前结论

这 20 篇样本给出的最重要启发，不是“顶级机构更会写话”，而是：

`它们在结论前多做了一层又一层审计。`

对本仓库而言，真正值得固化成技能的，不是它们的文风，而是它们的审计顺序。
