---

## 一、为什么要做 K 线节点硬数据

价格曲线只能解释产业和利润变量，但投资闭环必须回答：

> 股价到底什么时候启动、什么时候见顶、什么时候退潮？

因此每只标的都要补：

1. 主升启动日期；
2. 第一次顶部日期；
3. 二次顶部日期；
4. 退潮确认日期；
5. 阴影低点；
6. 前复权价格；
7. 区间涨跌幅；
8. 最大回撤；
9. 成交额和换手率；
10. 对应产业数据和财务数据。

## 三、统一字段

每张 K 线节点表必须包含：

| 字段 | 说明 |
|---|---|
| 节点类型 | 主升启动、第一次顶部、二次顶部、退潮确认、低点等 |
| 日期 | 精确到交易日 |
| 前复权价格 | 统一使用前复权口径 |
| 后复权 / 不复权价格 | 可选，用于核对 |
| 区间涨跌幅 | 与上一关键节点比较 |
| 最大回撤 | 从顶部到低点 |
| 成交额 | 判断资金参与度 |
| 换手率 | 判断拥挤度 |
| 对应价格变量 | 如碳酸锂、硅料、六氟、组件 |
| 对应财务变量 | 毛利率、利润、现金流、存货、减值等 |
| 当时市场叙事 | 新闻、政策、产业逻辑 |
| 事后验证 | 是否被后续数据验证或证伪 |
| 数据源 | 东方财富、Wind、同花顺、TradingView、Choice 等 |

## 四、第一批核心问题

锂资源双卡要验证：

> **碳酸锂价格顶部 → 赣锋 / 天齐股价顶部 → 毛利率顶部 → 利润顶部** 的先后顺序。

重点不是精确预测，而是建立产业周期规律：

1. 股价是否领先价格？
2. 价格是否领先利润？
3. 利润最好时估值是否已经不再扩张？
4. 锂价下跌后，存货和减值是否滞后暴露？
5. 资源股的低 PE 是否可能是周期利润陷阱？

## 六、数据口径要求

1. 股价统一前复权；
2. 所有节点必须标注数据源；
3. 如果使用不同软件，需要记录口径差异；
4. 不能用模糊表述代替数据；
5. 未核验字段必须写“待补”；
6. 每张表最终要和价格、财务、估值表联动。
