## 一、数据源探针与口径边界

### 1. 公募基金重仓

公募基金重仓主表已另立为 `14股公募基金重仓实填总表_第一版.md`。本版保留公募重仓的来源边界说明，但不再在此页展开数值。

东方财富 `RPT_STOCK_DETAILS_CHANGE` 重要机构明细可用于探针，但当前主要显示国家队、社保等重要机构线索，不能替代完整的公募基金重仓全口径。因此，上一版估值表中的“重要机构合计”只能作为资金线索，不能被写成公募拥挤度已经验证。

### 2. 北向持仓

北向持仓页面和报表名已从东方财富页面脚本中识别：

- 页面：`data.eastmoney.com/hsgtcg/list.html`
- 脚本：`/newstatic/js/hsgtcg/gglist.js?11111`
- 交易日列表报表：`RPT_MUTUAL_TRADEDATE`
- 个股北向持仓报表：`RPT_MUTUAL_STOCK_NORTHSTA`

交易日 API 返回的最新可见日期为 `HOLD_DATE=2026-06-25`。但在当前环境中，`RPT_MUTUAL_STOCK_NORTHSTA` 返回 `9701 服务器繁忙`，所以本版不填任何北向数值，只标注为待核验。

### 3. Wind / Choice

本地没有 Wind / Choice 终端数据，本版只能定义需要交叉校验的字段和状态。所有 Wind / Choice 校验项在本版均保留为 `待核验` 或 `待补`。

### 4. 本版边界

本版不能宣称资金拥挤度已闭环。它完成的是口径推进：把上一版笼统的公募、北向、Wind / Choice 待补，转换为公募重仓、北向持仓、Wind / Choice 三张来源明确的待补 / 待核验矩阵。

## 五、与上一版估值表的闭环关系

上一版 `14股估值与重要机构持仓实填总表_第一版.md` 已经完成三件事：

1. 用东方财富估值分析补齐 14 股峰值日、退潮确认日、阴影低点日的 `PE_TTM`、`PB_MRQ`、`TOTAL_MARKET_CAP`；
2. 用东方财富重要机构持股统计建立第一版“重要机构合计”线索；
3. 明确公募基金重仓、北向持仓、Wind / Choice 交叉校验仍是 `待补` / `待核验`。

本表不是推翻上一版，而是把上一版最后一张笼统待补矩阵拆成更清楚的执行表：

- 公募重仓：等待基金季报 / Wind / Choice / 东方财富基金持仓全口径；
- 北向持仓：已识别东方财富页面、脚本和报表名，但个股报表当前返回 `9701 服务器繁忙`；
- Wind / Choice：定义字段，等待终端或导出数据复核。

因此，估值压缩证据已经有东方财富口径的第一版闭环；资金拥挤度还没有闭环，只是从“笼统待补”推进为“来源明确、字段明确、接口状态明确”的待补矩阵。

## 六、阶段结论

本版完成的是工作台搭建，不是资金持仓数据实填。它把 14 股新能源样本的资金拥挤度补证拆成三条线：公募基金重仓、北向持仓、Wind / Choice 交叉校验。

结论边界必须保持清楚：不能把东方财富重要机构合计写成公募基金重仓，不能在 `RPT_MUTUAL_STOCK_NORTHSTA` 当前返回 `9701 服务器繁忙` 的情况下填北向数值，也不能在本地没有 Wind / Choice 终端数据时声称交叉校验完成。

下一步优先级是补公募基金季报明细与北向接口重试；完成后再回填资金拥挤度是否真正闭环。
