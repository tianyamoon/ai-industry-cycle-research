## 一句话结论

当前选股准绳不是把旧规则替换成技术图形，而是在原有交易纪律上增加技术面执行门槛。完整口径为：

`方向优先于个股 + 经营验证必须大于故事 + 高位强票不无脑追 + 低位方向不能因为没涨就买 + 必须有反方条件 + 买的是可验证的路径而不是概念标签 + 月线向上 + 周线向上 + 月线乖离率不过大`

其中前六条是底层选股纪律，`月线向上 + 周线向上 + 月线乖离率不过大` 是主动买入候选的技术图形硬门槛；日线只负责择时，不能替代月线和周线，也不能覆盖前六条纪律。

---

## 一、底层六条纪律

### 1. 方向优先于个股

先判断资金、产业变量和经营验证是否共同指向某个细分方向，再在方向内选代表标的。不能因为某只个股强，就反推整个方向已经成立。

### 2. 经营验证必须大于故事

方向入选必须能回到订单、收入、利润、OCF、回款、客户或价格变量。只有概念、政策想象或同业映射，不能直接写成主动买入理由。

### 3. 高位强票不无脑追，必须审慎评价

高位强票可以继续跟踪和持有审计，但新增仓位要看分歧承接、估值容错和成交拥挤。强趋势不是追高的充分理由。

### 4. 低位方向不能因为没涨就买

低位只能说明预期可能没打满，不能自动等于赔率好。若方向没有组级趋势、经营闭环和技术转强，只能列为观察池。

### 5. 必须有反方条件

每个候选方向都要写清楚什么情况说明判断错了，包括基本面证伪、价格承接失败、估值透支、组内分化或外部映射转弱。

### 6. 买的是可验证的路径，不是概念标签

每次买入前要说清楚买的是哪条传导路径：例如 `订单 -> 收入 -> 利润 -> OCF`，或 `价格上行 -> 毛利率改善 -> 利润兑现`。不能只写“AI算力”“国产替代”“AIDC”等标签。

---

## 二、技术图形硬门槛

### 1. 月线资格线

主动买入候选必须满足月线处于上升趋势，至少具备以下特征中的多数：

1. 月收盘仍在关键中长期均线之上，优先看 `6月 / 12月` 均线；
2. `6月` 均线走平向上，价格没有重新跌回长期下降通道；
3. 最近 `3-6` 个月的高点或低点至少有一项持续抬高；
4. 不处在连续月线阴跌、跌破平台后反抽或长期均线空头排列阶段。

若月线不是上升趋势，即便基本面很强，也只能放入 `研究池 / 观察池 / 持有审计`，不得列为新增买入候选。

### 2. 月线乖离率排除线

月线向上只代表趋势资格，不代表当前位置适合买入。若月线乖离率过大，说明价格已经远离中长期成本区，主动买入应降级为 `持有审计 / 等回踩 / 等平台消化`。

计算口径：

1. `6月线乖离率 = 当前价 / 6月均线 - 1`；
2. `12月线乖离率 = 当前价 / 12月均线 - 1`；
3. 若 7 月月K尚未收盘，用最新收盘价与动态月均线估算，并标注 `动态月线口径`。

初始纪律阈值：

1. `6月线乖离率 > 60%`，原则上不得新增主动买入；
2. `12月线乖离率 > 100%`，原则上不得新增主动买入；
3. `6月线乖离率在 40%-60%`，只能等分歧承接或平台消化后再评估；
4. 若标的是极高波动小票，即便未触发上述阈值，也要结合近 `3-6` 个月涨幅、成交拥挤和距阶段高点位置单独审计。

### 3. 周线执行线

在月线合格后，还必须确认周线处于上升趋势，至少具备以下特征中的多数：

1. 周收盘在 `10周 / 20周` 均线之上；
2. `20周` 均线走平向上，或价格正在有效突破周线平台；
3. 最近 `4-8` 周低点抬高，没有连续跌破关键平台；
4. 放量突破后没有快速跌回平台下方。

若月线合格但周线未转强，只能写成 `等待右侧确认`，不得执行主动买入。

### 4. 日线择时线

日线只用于决定买点，不用于决定资格：

1. 日线回踩不破周线结构，可以作为试错窗口；
2. 日线放量突破但周线和月线不合格，不得追；
3. 日线急涨若造成短期乖离过大，应先等承接，而不是把突破直接写成买入理由。

## 三、评分改造

后续日报的 `价格趋势 20%` 先通过技术图形硬门槛，再计算短中期强弱：

| 状态 | 处理 |
|---|---|
| 月线向上 + 周线向上 | 可以进入主动买入候选，再看基本面、估值和拥挤度 |
| 月线向上 + 周线向上 + 月线乖离率过大 | 不做新增主动买入，只能持有审计、等回踩或等平台消化 |
| 月线向上 + 周线未确认 | 只能等待右侧，不做新增主动买入 |
| 月线未向上 + 周线反弹 | 只能算反弹或修复观察，不写成主线接力 |
| 月线向下 + 周线向下 | 研究池或排除，除非是明确的左侧策略且另行标注 |

价格趋势分的使用纪律：

1. 月线不合格，价格趋势分原则上不得超过 `6/20`；
2. 周线不合格，价格趋势分原则上不得超过 `10/20`；
3. 月线乖离率过大，价格趋势分原则上不得超过 `12/20`，且不得写成新增主动买入；
4. 月线、周线均合格且乖离率不过大后，才允许用 `5日 / 20日 / 60日 / 距高点` 继续比较强弱；
5. 若月线和周线同时转弱，即使综合分暂时较高，也应从 `Top策略` 降为 `防守审计`。

## 四、选股顺序

后续选股按下面顺序，不跳步：

1. 先做细分方向排序，确认方向是否在前排；
2. 再确认买的是哪条可验证路径，而不是概念标签；
3. 再看经营验证：订单、收入、利润、OCF、回款是否形成闭环；
4. 再看月线是否向上，决定能否进入主动买入候选；
5. 再看月线乖离率是否过大，决定能否新增主动买入；
6. 再看周线是否向上，决定能否执行；
7. 再看估值赔率：价格里是否已经写满多年高增长；
8. 再看拥挤风险：成交、换手、距高点、回撤是否已经危险；
9. 再写清楚反方条件：什么情况说明这个方向或标的判断错了；
10. 最后写清楚加仓、减仓和退出条件。

---

## 五、降级纪律

出现以下任一情况，不能写成新增买入：

1. 月线下降趋势未扭转；
2. 周线仍在下降通道，只靠日线反弹；
3. 基本面有利好，但股价在周线级别不能承接；
4. 只有低位，没有月线和周线的上升结构；
5. 只有日线突破，没有周线确认；
6. 只有题材扩散，没有经营兑现；
7. 只有估值便宜，没有技术转强和基本面催化。
8. 只有概念标签，无法说清楚可验证路径；
9. 没有明确反方条件，无法判断何时认错。
10. 月线趋势虽向上，但 `6月线 / 12月线` 乖离率过大，新增买入已经变成追高而不是右侧进攻。

## 六、当前应用口径

1. `HBM / 半导体设备 / 先进封装`：即便仍是当前前排方向，也必须逐只补月线和周线趋势，不因方向强就自动通过。
2. `AIDC供配电 / AI算力基建大组`：即使经营线索改善，若月线和周线未形成上升趋势，仍只能作为观察池。
3. `IDC / AIDC运营`：若只出现日线反弹而月线、周线未修复，继续按修复观察处理。
4. `终端 / 具身智能 / 应用层`：短线转强但经营证据不足时，不得用日线强势绕过月周线硬门槛。
5. 技术图形不能覆盖底层六条纪律：月线和周线合格，但方向不在前排、经营验证不足、估值透支、月线乖离率过大或反方条件不清，仍不得列为主攻方向。

## 九、对用户输出格式

当用户要求选择方向、个股或周一进攻方向时，输出不得只给结论，必须同时写清楚逻辑链。

### 1. 每个方向必须说明七项

1. `方向地位`
   - 说明该方向在当前排序中是主攻、备选、观察还是回避。
2. `经营验证`
   - 说明当前证据来自订单、收入、利润、OCF、回款、客户、价格，还是只有故事。
3. `位置与拥挤`
   - 说明是高位强趋势、低位未启动、修复反弹，还是高位分化。
4. `低位纪律`
   - 如果方向低位，必须说明为什么不能仅因没涨就买。
5. `反方条件`
   - 说明什么情况会证明该方向判断错了。
6. `可验证路径`
   - 说明买的具体传导路径，例如 `订单 -> 收入 -> 利润 -> OCF`，不能只写概念标签。
7. `技术门槛`
   - 说明月线、周线、月线乖离率是否满足；若待核，必须显式写 `月周线待核 / 月线乖离率待核`。

### 2. 每个个股必须说明四项

1. `角色`
   - 主锚、补强位、弹性票、平台票、旁证票、观察票或反证票。
2. `为什么看`
   - 它验证了方向里的哪个变量。
3. `为什么不无脑买`
   - 估值、位置、拥挤、现金流、AI纯度、订单兑现或技术形态的限制。
4. `触发 / 证伪`
   - 什么情况下允许进攻，什么情况下必须放弃或降级。

### 3. 结论表达顺序

推荐输出顺序为：

1. 先给一句话结论；
2. 再给方向总排序；
3. 再逐方向解释七项条件；
4. 再逐个股解释角色与限制；
5. 最后给周一执行条件和反方撤退条件。
