# AI应用方向行情与经营跟踪看板
> 所属模块：03_结果层/01_总表整合与AI映射
> 生成日期：`2026-07-04`
> 看板定位：把应用层从“方向稿 + 单卡 + 估值散点”压缩成可直接维护的结果层总面板。
> 使用边界：本看板优先承接桥接层应用总表、应用组单卡和结果层估值总表；若后续没有新的正式经营披露，不强行迁移阶段。

---

## 一句话结论

AI 应用层当前最稳的正式口径不是“应用全面爆发”，而是：`办公协同与金融入口` 已进入成熟兑现审计，`AI教育场景硬件入口` 已进入公告级兑现验证，`综合平台与企业软件` 仍在“逻辑成立、经营继续验证”，`AI原生创意工具与数据智能转型` 仍偏模式切换观察，`端侧 Agent / 机器人 / AI玩具` 依然只能放在观察层。

## 二、当前市场在交易什么

### 1. 已在交易的

1. `金山办公`
   - 企业协同与 AI 办公的真实付费验证。
2. `同花顺`
   - 高入口、高频数据服务叠加 AI 的经营杠杆。
3. `科大讯飞`
   - 大模型平台、MaaS 和行业应用的中长期平台位次。
4. `视源股份`
   - 教室智能终端、希沃教育 AI 产品和智能控制部件共振下的公告级利润验证。

### 2. 还在继续验证的

1. `用友网络`
2. `汉得信息`
3. `万兴科技`
4. `拓尔思`

这些方向还不能只用“有产品”或“有合同”来长期化，必须继续回到收入、扣非和 OCF。

### 3. 市场最容易过度交易的

1. `端侧 Agent / AI眼镜`
2. `具身 / 机器人应用`
3. `AI玩具 / 陪伴硬件`

原因不是这些方向不重要，而是它们当前更缺正式经营闭环。

## 三、应用层切换观察板

| 观察问题 | 当前答案 | 升级条件 | 降级条件 |
|---|---|---|---|
| 应用层是否已普遍进入兑现期 | 还没有，只是部分龙头进入兑现审计 | 企业软件和垂类应用也出现收入、利润、OCF 同步强化 | 只有头部样本能兑现，其他继续停留在题材或高投入阶段 |
| AI 教育硬件入口是否可升级为成熟兑现 | 刚进入公告级兑现验证，但仍需拆分 AI 教育与硬件/非经常性收益 | 2026H1 教育品牌收入、AI 教育付费、合同负债和 OCF 同步改善 | 高增主要来自非经常性收益、控制部件或备货周期，AI 教育收入无法拆分 |
| 企业软件层是否开始整体修复 | 还在早期验证 | 用友、汉得等持续看到 AI 收入占比提升与 OCF 修复 | 只有签约，没有收入和现金流改善 |
| 入口型应用是否最先穿越 | 当前是 | 景气回落后收入和合同负债仍有韧性 | 高活跃度一旦回落，增长明显失速 |
| 端侧 Agent 是否已进入结果层主线 | 还没有 | 出现量产、订单、收入占比与订阅绑定披露 | 继续只有发布、展会和渠道热度 |
| 具身应用是否可上收正式结论 | 还不行 | 上市公司正式订单和收入开始稳定披露 | 继续停留在映射交易与合作问答 |
| AI玩具是否可上收正式结论 | 还不行 | 出货、复购、订阅和渠道动销开始稳定披露 | 继续只有社交传播与短期热卖描述 |

## 四、当前最重要的三条判断

### 1. 应用层已经分层，不再是一条统一贝塔

当前至少应拆成：

1. 已兑现审计：`金山办公 / 同花顺`
2. 兑现验证：`视源股份`
3. 高投入验证：`科大讯飞 / 用友网络 / 汉得信息`
4. 模式切换观察：`万兴科技 / 拓尔思`
5. 主题观察：`端侧 Agent / 机器人 / AI玩具`

### 2. 应用层的核心变量不是“用户热度”，而是“付费 -> 收入 -> 现金流”

尤其后续最该补的是：

1. 续费
2. AI 收入占比
3. 合同负债
4. 扣非利润
5. OCF

### 3. 结果层后续必须把应用看板纳入周更

下次再做 AI 周更时，不能只跟踪：

1. 硬件主线
2. AIDC供配电
3. 资源运营
4. 液冷

还必须同步回答：

1. 应用成熟层是否继续强化
2. 企业软件层是否出现新的经营修复
3. 端侧 Agent / 机器人 / AI玩具是否仍停留在观察层

## 五、下一轮补证优先级

1. `最高优先`
   - 金山办公、科大讯飞、同花顺、视源股份
   - 目的：确认成熟应用层的付费、合同负债、现金流和估值久期是否继续自洽。
2. `第二优先`
   - 用友网络、汉得信息、万兴科技、拓尔思
   - 目的：确认企业软件与垂类应用是否开始从“有产品”走向“有收入和 OCF”。
3. `第三优先`
   - 端侧 Agent / AI眼镜 / 具身应用 / AI玩具
   - 目的：只在出现量产、出货、订单、收入占比或订阅复购时升级，不因热度强行上收。
4. `候选池补证`
   - 医疗AI / AI制药、营销电商 / 外贸、内容IP / 游戏、数据安全 / 版权治理、工业 / 政务 / 交通、视觉 SDK / 端侧软件
   - 目的：结果层不把这些方向直接写成成熟兑现，但周更必须检查桥接层 L0/L1 候选池，防止应用层覆盖继续偏少。

## 五点五、结果层不穷尽标的

当前看板中的公司是 `可进入经营审计的代表样本`，不是 AI 应用方向的全量候选。全产业链中市值不低于 `100亿元` 的已路由 AI 标的已进入桥接层强制观察池；应用只是其中一个子集，且不会因为市值较大而自动进入结果层。更宽的候选池已回写到：

1. [AI应用方向细分框架](../../02_桥接层/02_AI产业链研究卡/AI应用方向细分框架.md)
2. [AI应用方向滚动跟踪总表](../../02_桥接层/02_AI产业链研究卡/AI应用方向滚动跟踪总表.md)
3. [AI全景细分样本宇宙总表](../../02_桥接层/02_AI产业链研究卡/AI全景细分样本宇宙总表.md)

结果层只在候选样本出现 `付费 / 合同 / 续费 / 收入 / 扣非 / OCF / 出货 / 订阅` 中至少一条正式强化后，才把它从候选池迁入看板。

市值观察池和全 A 路由审计的完整代码、业务纯度和第一变量，以 [AI全景细分样本宇宙总表](../../02_桥接层/02_AI产业链研究卡/AI全景细分样本宇宙总表.md) 及其数据附件为唯一明细入口；本看板只维护代表样本和状态迁移。
