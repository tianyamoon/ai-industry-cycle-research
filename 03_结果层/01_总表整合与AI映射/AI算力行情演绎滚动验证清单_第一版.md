> 所属模块：03_结果层/01_总表整合与AI映射  
> 生成日期：2026-07-02  
> 使用目的：把 `AI算力行情演绎跟踪表_第一版.md` 后续需要周度更新的 `短线斜率 / 中线强弱 / 高位回撤 / 财务验证 / 接力触发器` 压成统一执行清单。  
> 使用边界：本清单服务“行情演绎到了哪一步”的持续跟踪，不新增无来源传闻结论，不把单日波动直接上升为阶段反转。

## 零、技术图形硬门槛

从 `2026-07-05` 起，周更和日报新增一条主动买入硬门槛：

`月线向上 + 周线向上`

执行口径：

1. 月线决定能否进入主动买入候选；
2. 周线决定能否执行；
3. 日线只负责择时；
4. 月线或周线不满足时，即便基本面和题材强，也只能写成 `观察 / 等待右侧 / 持有审计`；
5. 若只有日线反弹，不能把细分或个股从观察位升级为接力位。

## 二、分细分执行要点

### 光模块 / 高速互联

1. 优先盯 `中际旭创`、`新易盛` 是否重新带动组内斜率转强。
2. 看 20 日涨幅是否从当前弱化状态回到明显正值。
3. 若距60日高点回撤继续扩大，则从“主升中后段”降级到“高位分化审计”。

### PCB / CCL / 高速互连辅材

1. 优先盯 `沪电股份` 与 `生益科技` 是否继续维持强势。
2. 看分化是否继续扩大，还是重新回到主线内部收敛。
3. 必须把 CapEx、毛利率和产能利用率放在一起审计，不能只看涨幅。

### HBM / 高性能存储代理

1. 优先盯 20 日涨幅是否持续高于 `30%`。
2. 看 `澜起科技` 是否继续创新高，`佰维存储`、`江波龙` 的现金流是否跟上利润。
3. 若出现高位放量回撤且距高点跌破 `-15%`，则先按拥挤退潮处理。

### 先进封装 / 封测 / 材料

1. 优先盯 `长电科技` 与 `华海诚科` 是否还能共同维持强势。
2. 看平台龙头和小盘材料是不是同步，避免只剩小票冲刺。
3. 收入、AI相关占比和估值容错必须一起看。

### 国产AI芯片 / 平台

1. 优先盯 `寒武纪`、`海光信息` 的斜率和成交强度。
2. 看芯片核心资产是否继续跑赢平台承接资产。
3. 若高估值不能被利润斜率承接，应先降级为高位兑现审计。

### AI服务器整机 / ODM

1. 优先盯 `工业富联` 的斜率是否重新转强。
2. 看收入能否重新跟上利润改善，避免只剩低毛利制造估值上移。
3. 若弱于指数且毛利率无改善，则维持底座而非主线处理。

### AIDC供配电 / UPS / 变压器

1. 优先盯 `科华数据`、`科士达` 的20日与60日斜率。
2. 同步盯 `金盘科技`、`科华数据`、`科士达` 三条内部路径的兑现顺序。
3. `金盘科技`
   - 看 `订单 -> 合同负债 / 存货 -> 收入 -> OCF` 是否闭环。
4. `科华数据`
   - 看 `数据中心分部收入 -> 平台交付 -> 应收效率 -> OCF` 是否继续强化。
5. `科士达`
   - 看 `数据中心核心业务收入 -> 标准化产品放量 -> AI 纯度拆分 -> OCF` 是否逐步清晰。
6. 只有当行情和报表同步改善，且三条路径中至少一条出现连续强化，才能从低位观察升级为补涨加速。

### AI算力基建大组观察池

1. 先看 `核心供配电` 是否仍是经营锚，而不是被低 AI 纯度电力设备旁证替代。
2. 再看 `液冷 / 温控` 是否从单点强势扩散到两只以上同步改善。
3. 再看 `IDC / AIDC运营` 是否出现上架率、利用率和 OCF 的统一修复。
4. 若大组行情转强但核心供配电三路径仍没有 `订单 / 收入 / OCF` 闭环，只能写成 `基建共振观察`，不能写成 `AIDC供配电接力确认`。

### 液冷 / 温控

1. 优先盯 `申菱环境` 的强势是否能扩散到组内。
2. 看 `英维克`、`高澜股份` 是否从弱势转向跟涨。
3. 若只有单一标的维持强势，组级结论不升级。

### IDC / AIDC运营 / 算力租赁

1. 优先盯 `润泽科技` 是否能维持相对强势。
2. 看组内 `60日上涨占比` 能否从 `16.67%` 修复到 `50%` 以上。
3. 看上架率、利用率和 OCF 是否共同改善，而不是只剩单点题材反弹。

## 三、组级观察原则

### 判断

1. 当前 AI 算力行情的主线在 `硬件`，不在 `基建`。
2. `HBM / 先进封装` 当前更适合按 `高位加速 -> 拥挤审计 -> 回撤验证` 路径跟踪。
3. `PCB / 光模块` 当前更适合按 `主升后分化 -> 龙头再确认` 路径跟踪。
4. `供配电 / 液冷` 当前更适合按 `低位观察 -> 是否形成接力` 路径跟踪，其中 `供配电` 必须再拆成 `订单型 / 平台型 / 标准化` 三条内部路径。
5. `IDC / AIDC运营` 当前必须按 `退潮 -> 修复 -> 再确认` 路径跟踪，不能直接假设主线回归。
6. `AI算力基建大组` 只作为跨层共振观察口径；它能帮助判断资金是否从硬件外溢到基建，但不能替代三路径经营闭环。
7. 新增买入候选必须先通过 `月线向上 + 周线向上`，不再只看 20 日和 60 日斜率。

## 四、当前动作建议

1. `第一优先`：每周更新 `HBM / 先进封装` 的 20 日涨幅和距60日高点回撤，判断高位加速是否结束。
2. `第二优先`：继续跟 `光模块 / PCB` 的龙头是否出现二次转强，避免把滞涨误当成新主升。
3. `第三优先`：继续跟 `AIDC供配电` 的低位抬升信号，并同步判断 `AI算力基建大组` 是否出现供配电、液冷、IDC运营和电力设备旁证的跨层共振。
4. `第四优先`：把 `液冷` 作为个股强势而非组级主线处理，直到出现组内扩散。
5. `第五优先`：把 `IDC / AIDC运营` 继续放在风险补位和修复观察层，不提前升级。

## 五、周更最小更新集

每次周更至少更新以下字段：

1. `5日涨跌幅`
2. `20日涨跌幅`
3. `60日涨跌幅`
4. `距60日高点`
5. `组内上涨占比`
6. `组内最强标的是否变化`
7. `阶段标签是否变化`
8. `对应财务或订单验证是否出现`
9. `AIDC供配电` 内部三条路径的第一变量是否变化
10. `AI算力基建大组` 四个子层是否出现两个以上同步改善
11. `月线趋势` 是否仍向上
12. `周线趋势` 是否仍向上

若只更新价格、不更新触发条件和验证项，这张清单就会重新退化成普通涨跌幅表。

## 六、本版结论

这张清单的核心目的，不是告诉我们“哪只今天更强”，而是让我们持续回答三个问题：

1. `主线有没有从硬件切到基建？`
2. `高位加速是不是开始转成高位分化？`
3. `哪些看起来还没涨的方向，是真低位预期差，哪些只是没资金要？`

当前答案是：

> 主线仍在硬件，高位最强的是 `HBM / 先进封装`，最值得继续观察接力的是 `供配电`，最需要保持克制的是 `IDC / AIDC运营`。
