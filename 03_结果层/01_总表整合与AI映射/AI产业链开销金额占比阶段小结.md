# AI产业链开销金额占比阶段小结

> 所属模块：03_结果层/01_总表整合与AI映射
> 生成日期：2026-07-01
> 研究定位：把“AI 产业链开销金额占比”从框架与待补表，压缩成一份当前可正式引用的阶段小结。
> 口径说明：本版完成标准是“足以支撑相对顺序判断与正式引用”，不是“所有细分都已有精确百分比”。

---

## 一句话结论

AI 产业链的“开销金额占比”研究，当前已经可以正式支撑四层判断：训练型 `CapEx` 的最大钱包仍集中在 `芯片 / 存储 / 服务器集群`，推理型 `Opex` 的最大钱包仍集中在 `Token 推理成本 + 平台入口 + 云资源承接`，企业 AI 项目的真实预算往往先流向 `实施交付 + 知识库 / 数据治理 + 私有化部署`，而不是天然先流向 `模型授权本身`；`终端 / 具身智能` 这一层也已经从待补章节升级为独立结果层总表，当前最稳的正式判断是 `整机硬件先吃总钱包，端侧 SoC / 感知器件更接近厚利润池，订阅 ARPU 仍主要是后装增量钱包`。

---

## 一、研究边界

### 1. 研究对象

本阶段小结覆盖的是 AI 产业链中“谁吃到总钱包、谁吃到增量钱包、谁吃到利润池”的结构问题，不讨论单一公司估值高低，也不等同于股价弹性排序。

### 2. 当前覆盖的四类钱包场景

1. `训练型 CapEx 钱包`
2. `推理型 Opex 钱包`
3. `企业 AI 项目预算钱包`
4. `终端 / 具身智能 BOM 钱包`

### 3. 当前证据边界

本版阶段判断主要依赖三类正式来源：

1. 海外大厂 earnings call 与资本开支披露；
2. 官方 API / 平台定价页；
3. 政企 / 教育 / 医疗等正式项目公告、采购意向与绩效报告。

本版明确不采用：

1. 无出处的行业传闻；
2. 不能区分收入占比和成本占比的二手总结；
3. 只讲“趋势”但没有预算口径的泛化观点。

---

## 二、当前已确认的核心事实

### 1. 训练型 CapEx 已能确认相对顺序

1. Alphabet 2026Q1 已给出 `技术基础设施投资约 60% 在服务器、40% 在数据中心和网络设备` 的结构锚。
2. Microsoft FY2026Q3 已给出 `当季约三分之二 CapEx 属于短寿命资产，主要是 GPU 和 CPU` 的正式口径。
3. Micron Fiscal Q3 2026 已给出 `数据中心收入年化 run rate 超过 1000 亿美元` 与 AI 拉动存储紧张的利润侧锚点。

当前可直接引用的事实口径是：

> 训练型钱包已经不再只是“芯片最重要”的抽象判断，而是已有正式口径支持 `芯片 / 存储 / 服务器集群` 属于大钱包，`园区 / 供配电 / 温控` 属于长寿命重资产承接。

### 2. 推理型 Opex 已能确认计费结构

1. OpenAI 官方定价页已经把 `input / cached input / output` 分开计费。
2. Gemini 官方定价页已经把 `context caching`、`storage price`、`grounding with Google Search` 单列收费。
3. Anthropic 官方定价页已经把 `input / output / prompt caching write / read` 分开计费。
4. AWS Bedrock 官方定价页已经把 `Standard / Flex / Priority / Reserved` 等服务层级单列。

当前可直接引用的事实口径是：

> 推理型钱包已经不再只是“模型调用费”的单层结构，而是至少拆成 `推理成本 + 平台入口 + 缓存 / 存储 + 工具 / 搜索 / 服务等级` 四段持续 Opex。

### 3. 企业 AI 项目预算已能确认真实落地方向

1. 中国信达正式把项目拆成 `大模型及知识库开发实施` 与 `算力租赁服务` 两包，并明确 `本期无大模型采购费用`。
2. 常州一院 500 万元项目、深圳医保 30 万元项目、西安石油大学 220 万元项目、中国矿业大学 9.1 万元项目，已经覆盖 `医疗 / 政务 / 教育 / 部门级轻量知识库` 四类落地场景。
3. 海淀区 382.78 万元智能客服绩效报告，已给出最强反证：`人工坐席服务 364.08 万元`，而 `专业大模型 5.8 万元`。

当前可直接引用的事实口径是：

> 企业 AI 项目在真实预算里，常常先买 `实施、知识库、调优、集成、私有化部署、运营服务`，模型本身未必是最大钱包。

### 4. 终端 / 具身智能已从“第一批锚点”升级到“可正式引用的一版总表”

1. Meta AI 眼镜、Galaxy S25、Unitree G1 / R1 / Go2 已经给出第一批官方整机价格锚。
2. iFixit 已经给出 AI 眼镜与 AI 手机的第一批拆机 / 芯片识别锚点。
3. ChatGPT Plus、Google AI Pro、Meta One Premium 已经给出第一批消费侧订阅价格锚。
4. [AI终端_具身智能整机BOM与订阅ARPU对照总表_第一版](</F:/股票/ai-industry-cycle-research/03_结果层/01_总表整合与AI映射/AI终端_具身智能整机BOM与订阅ARPU对照总表.md>) 已把整机价格、订阅年化 ARPU、端侧芯片锚与机器人感知器件锚压成统一结果层入口。
5. 但 AI 手机、AI 眼镜、机器人整机 BOM、端侧 SoC 与持续订阅 ARPU 仍缺更连续、更可比的精确结构口径。

当前可直接引用的事实口径是：

> 终端和具身智能的“钱包迁移”方向已经明确，而且已经可以正式写成：`当前第一笔大钱包先落整机硬件，端侧 SoC / 内存 / 感知器件更接近利润池，订阅 ARPU 仍主要是后装增量钱包`；但暂时还不该写成精确 BOM 百分比结论。

本轮补证后，可以再补一层更硬的年化订阅锚：

1. ChatGPT Plus 当前公开月费为 `$20 / 月`，折算约 `$240 / 年`；
2. Google AI Pro 当前公开月费为 `$19.99 / 月`，折算约 `$239.88 / 年`；
3. 对比之下，`Galaxy S25` 当前公开价 `$799.99`，约等于 `3.3 年` 主流消费级 AI 订阅；
4. `Unitree Go2`、`R1`、`G1` 的整机价格更分别约等于 `6.7 年`、`20.4 年`、`56.3 年` ChatGPT Plus。

这说明当前终端 / 具身智能的钱包结构，至少在消费和早期机器人阶段，仍然显著偏向 `硬件前装大钱包`，而不是 `订阅即时压过整机`。

## 三、阶段判断

### 1. 这条研究线已经从“框架搭建期”进入“证据累积期”

当前已经不是在回答“要不要研究钱包份额”，而是在回答：

1. 哪些相对顺序已经足够正式引用；
2. 哪些环节已有硬证据，哪些仍停留在待补；
3. 哪些判断可以写进正式总论，哪些只能留在待补表。

### 2. 训练、推理、企业项目三张表已基本具备正式引用能力

1. `训练型 CapEx`
   - 已能写相对顺序
   - 仍不能写统一百分比
2. `推理型 Opex`
   - 已能写计费结构
   - 仍不能写统一行业均价
3. `企业 AI 项目预算`
   - 已能写真实预算主次
   - 仍需继续补更细分行业与更多项目拆分

### 3. 终端 / 具身智能处于“第一批锚点已出现，但数字仍滞后”

当前最合适的口径不是直接下结论，而是保留：

1. `整机 BOM` 可能成为总钱包中心；
2. `端侧 SoC` 更可能成为厚利润池；
3. `Agent / 云侧订阅` 更可能成为持续收费迁移方向。
4. 当前已经补到第一批 `年化订阅 ARPU` 对照，并已单独上收结果层总表；但在没有更多拆机与持续订阅 ARPU 口径前，仍不宜把它写成成熟钱包结构。

## 四、四条当前可正式引用的主结论

### 结论一：训练型 CapEx 的总钱包中心仍在上游算力链

#### 事实

1. Microsoft 正式披露短寿命资产以 `GPU / CPU` 为主。
2. Alphabet 正式披露服务器投资权重高于数据中心和网络设备。
3. Micron 正式披露 AI 正持续拉高数据中心存储景气。

#### 判断

训练型 `CapEx` 当前最可靠的相对顺序仍是：

`芯片与存储 > 服务器与集群 > 高速互联 > 数据中心配套 > 数据与安全`

#### 当前边界

这条结论目前能支撑相对顺序，不能支撑跨厂统一精确比例。

---

### 结论二：推理型 Opex 的大头不只是模型调用，而是按层拆分的持续运营费

#### 事实

1. 主流平台都已按 `input / output / cache` 分开计费。
2. 搜索、grounding、priority、reserved capacity 等平台层费用已被单列。
3. caching storage 已经把推理期预算继续沉到基础设施层。

#### 判断

推理型 `Opex` 当前最可靠的顺序不是单一“模型费”，而是：

`推理 Token 成本 + 平台入口 + 云资源承接`

共同争夺预算。

#### 当前边界

这条结论目前能支撑结构判断，不能支撑统一行业均价。

---

### 结论三：企业 AI 项目的最大钱包往往先流向交付和场景，而不是模型本身

#### 事实

1. 金融项目已经把 `实施 / 知识库 / 算力租赁` 分包。
2. 医疗、教育、政务、部门级系统都已出现 `知识库 + 集成 + 调优 + 私有化部署` 的组合预算。
3. 海淀案例明确给出 `模型本身显著小于人工服务与运营服务` 的预算拆分。

#### 判断

企业 AI 项目当前最可靠的预算主次是：

`应用改造 / 实施交付 + 数据治理 / 知识库 + 私有化部署`

优先于：

`模型授权 / MaaS`

#### 当前边界

这条结论目前已经有跨行业正式样本支撑，但仍需继续补更多中大型项目的细分拆账。

---

### 结论四：模型能力强，并不自动等于它拿走最大钱包

#### 事实

1. 信达项目明确 `本期无大模型采购费用`。
2. 海淀项目里 `专业大模型` 远小于 `人工坐席服务`。
3. 多个场景都把预算沉到平台、交付、服务和运营上。

#### 判断

AI 产业链研究如果只看“谁技术最强、谁卡脖子、谁最热”，很容易高估模型本身在真实预算中的占比，低估 `实施、交付、服务、运营、私有化` 这些慢变量钱包。

#### 当前边界

这条结论当前更适用于企业与公共服务场景，对终端消费级场景仍不能直接套用。

---

## 五、当前不能写得过满的部分

以下内容当前仍不能写成“已经闭环”：

1. 训练型 `CapEx` 各环节统一百分比；
2. 推理型 `Opex` 的统一单位成本；
3. 企业 AI 项目中 `安全 / 合规 / 审计` 的独立预算占比；
4. 终端 / 具身智能 BOM 的精确排序和持续订阅结构。
5. AI 眼镜、AI 手机、机器人之间可比的单机 BOM 拆解口径。

这些缺口不阻塞当前三条主链的正式引用，但会直接影响后续从“相对顺序”升级到“更细数字化结构”。

## 六、资本强度、回报质量与证伪口径

本次升级后，本阶段小结新增一个约束：钱包份额判断必须同时回答“这笔钱有没有形成好回报”。因此，后续不能只写谁吃到预算，还要写谁承担资本强度、谁获得利润池、谁能用现金流证明回报质量。

| 场景 | 资本强度 | 回报质量当前判断 | 最关键证伪指标 | 当前边界 |
|---|---|---|---|---|
| 训练型 CapEx | 高 | 芯片 / HBM / 高速互联更接近厚利润池，园区 / 供配电 / 温控更接近慢兑现承接 | 云厂 CapEx 放缓、硬件折旧周期缩短、毛利率下行 | 可写相对顺序，不能写统一百分比 |
| 推理型 Opex | 中 | 平台入口、缓存、搜索、服务等级可能提升利润池，但单位 Token 成本仍会持续竞争 | API 降价、缓存 / 存储成本下降、平台抽成被压缩 | 可写计费结构，不能写统一行业均价 |
| 企业 AI 项目预算 | 中 | 实施交付、知识库、私有化部署先吃预算，但利润率取决于可复制性和续费 | 项目验收失败、续费不足、回款拖延、实施毛利率下行 | 已有跨行业案例，但行业拆账仍需增加 |
| 终端 / 具身智能 BOM | 中高 | 整机先吃总钱包，SoC / 内存 / 感知器件更可能吃厚利润池，订阅仍属增量钱包 | BOM 降价、订阅转化率不足、出货量不达预期 | 独立结果层总表已形成，精确结构仍待补 |
| AIDC / 资源运营 | 高 | 方向成立不等于回报质量成立，必须用利用率、租赁价格、折旧、融资成本和 OCF 审计 | 利用率不升、租金下行、折旧 / 利息吞噬利润、OCF 弱于净利润 | 只能作为资本周期重点审计层 |
| 供配电 / 液冷 | 中到高 | 项目制弹性存在，但订单必须转收入、毛利率、回款和 OCF | 订单强但收入确认慢，回款和 OCF 不跟，毛利率回落 | 可作为接力观察，不宜直接写穿越 |

### 这一节带来的结论升级

1. 钱包研究从 `谁拿钱` 升级为 `谁拿钱、谁赚钱、谁把钱变成现金流`。
2. 对高资本强度环节，`总钱包大` 反而可能意味着折旧、融资和回款风险更高。
3. 对轻资产应用和平台入口，`资本强度低` 不自动等于高回报，还必须看续费、ARPU、客户留存和实施可复制性。

## 七、下一步建议

### 1. 结果层优先动作

下一步最适合进入结果层的，不是继续平铺更多零散案例，而是沿两条线推进：

1. 把本阶段小结和待补表一起作为正式总论的附属入口；
2. 在补出终端 / 具身智能连续 BOM 与更多订阅 ARPU 证据后，再写第二版阶段小结。

### 2. 补证层固定顺序

建议后续补证顺序固定为：

1. `终端 / 具身智能 BOM` 的连续拆机与更长时序 ARPU
2. `训练型 CapEx` 的先进封装与液冷价值量
3. `企业 AI 项目预算` 的安全 / 合规预算科目
4. `推理型 Opex` 的可比单位成本与平台抽成结构

---

## 八、本版结论

AI 产业链“开销金额占比”研究，现在已经从“先把框架搭起来”进入“训练、推理、企业项目三张子表可正式引用，终端 / 具身智能已拿到第一批锚点但仍待进一步补硬证据”的阶段。

当前最重要的动作，不是急着把所有环节写成精确百分比，而是先把已经成立的相对顺序和预算主次写实，再用最少量、最高价值的补证，把 `终端 / 具身智能 BOM` 这块补到可以正式上收的程度。
