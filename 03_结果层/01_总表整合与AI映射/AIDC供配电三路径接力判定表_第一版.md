## 一句话结论

截至 `2026-07-01`，`AIDC供配电` 仍处在 `未充分演绎 / 低位观察` 阶段，但已经不能再被写成单一设备链。更准确的写法是：`金盘科技` 代表 `订单型高弹性设备`，`科华数据` 代表 `平台型智算供电基础设施`，`科士达` 代表 `标准化稳态兑现`。当前是 `经营层先验证、行情层后确认`，下一步真正要跟踪的不是“供配电行不行”，而是三条路径里谁先把 `订单 / 收入 / OCF` 推成连续共振。

## 一、为什么要单独立这张表

### 事实

1. 结果层已经确认 `AIDC供配电` 是算力基建里最值得继续补证的接力候选，但行情仍未确认。
2. 桥接层 `算力设备组` 已完成三卡闭环，内部角色已经清楚分化。
3. `AI算力行情演绎周更看板_第一版.md` 与 `AI算力行情演绎滚动验证清单_第一版.md` 已把 `AIDC供配电` 明确拆成三条路径。
4. 截至 `2026-07-01`，组级行情仍偏弱：
   - `5日均值`：`1.02%`
   - `20日均值`：`-4.99%`
   - `60日均值`：`9.40%`
   - `YTD均值`：`2.34%`
5. 组级经营又并不弱：
   - `科华数据`：`2026Q1 收入 +17.57% / 归母 +13.15% / OCF +160.80%`
   - `金盘科技`：`2026Q1 新签订单 +64.48% / 数据中心订单 +278.45% / OCF -1.67亿元`
   - `科士达`：`2026Q1 收入 +31.47% / 归母 +24.84% / OCF +186.00%`

### 判断

`AIDC供配电` 当前最容易被误判的地方，不是方向错了，而是把不同兑现路径硬写成同一类资产，导致：

1. 把 `订单斜率` 误当成 `利润兑现`;
2. 把 `平台能力` 误当成 `已确认接力`;
3. 把 `稳态兑现` 误当成 `纯 AI 高弹性`;
4. 把 `经营已验证` 误当成 `行情已确认`。

## 三、当前更精确的阶段判断

### 1. 组级阶段

当前 `AIDC供配电` 的统一标签仍应维持：

`未充分演绎 / 低位观察`

原因不是经营差，而是：

1. 组级 `20日均值` 仍为负；
2. 组级 `60日均值` 只有 `9.40%`，远未达到结果层定义的 `30%` 接力升级线；
3. 三条路径都还没有在 `订单 / 收入 / OCF` 上形成持续、可被市场统一追认的共振。

### 2. 内部强弱顺序

若按当前已确认信息排序，更稳妥的写法是：

1. `经营平衡度`
   - `科华数据 ≈ 科士达 > 金盘科技`
2. `订单弹性`
   - `金盘科技 > 科士达 ≈ 科华数据`
3. `当前纯 AI 解释力度`
   - `金盘科技 > 科华数据 > 科士达`
4. `当前现金流审计压力`
   - `金盘科技 > 科华数据 > 科士达`

### 3. 当前不能下的结论

当前不能直接写：

1. `供配电已经是下一轮主线`
2. `三家公司都会同步受益`
3. `只要方向对，行情自然会补涨`
4. `经营验证已经等于股价确认`

## 四、扩组后的使用边界

### 1. 窄组与大组的分工

桥接层已新增 `AI算力基建大组扩展观察池_第一版.md`，把研究范围扩成：

`核心供配电 -> 电力设备旁证 -> 液冷 / 温控 -> IDC / AIDC运营`

但本文仍只负责 `AIDC供配电` 三路径判断：

1. `科华数据 / 金盘科技 / 科士达` 仍是窄口径核心样本；
2. 扩展观察池用于判断 `AI算力基建大组` 是否共振；
3. 电力设备、液冷和 IDC / AIDC运营样本只能作为旁证，不能直接替代三路径里的 `订单 / 收入 / OCF` 经营闭环；
4. 若大组走强但三路径没有经营闭环，本文仍不得把 `AIDC供配电` 上调为接力主线。

### 2. 什么时候大组信号有效

扩组信号只有在以下条件下才对本文构成加分：

1. 核心供配电组三只里至少两只同步转强；
2. 液冷 / 温控或 IDC / AIDC运营中至少一个子层出现经营层验证；
3. 电力设备旁证出现 AIDC / 数据中心相关订单、收入或交付口径，而不是只靠泛电网景气；
4. 核心供配电三路径至少一条出现 `订单 / 收入 / OCF` 连续强化。

### 3. 什么时候大组信号无效

以下情况不能用来上调本文判断：

1. 只有电力设备旁证强，但无法拆出 AIDC 或数据中心口径；
2. 只有液冷单一个股强，组内没有第二个经营锚点；
3. IDC / AIDC运营股反弹，但上架率、利用率、租赁价格和 OCF 没有改善；
4. 核心供配电三只仍弱，而大组里低 AI 纯度样本走强。

## 五、三路径各自的升级条件

### 1. 金盘科技：订单型高弹性设备

升级为“可正式写进接力主线”的条件：

1. 数据中心订单继续高增，且收入增速开始明显追上订单斜率；
2. `OCF` 由负转正并保持改善；
3. 存货、合同负债与应收不出现失控抬升。

证伪条件：

1. 订单继续强，但收入和利润传导停滞；
2. `OCF` 连续承压；
3. 海外交付或回款拖累兑现。

### 2. 科华数据：平台型智算供电基础设施

升级为“可正式写进接力主线”的条件：

1. 数据中心分部收入继续增长，并能稳定拆出产品 / 集成 / IDC 服务结构；
2. `收入 + 利润 + OCF` 继续同步改善；
3. 应收、库存、在建工程和合同负债维持在可控区间。

证伪条件：

1. 平台故事延续，但分部收入传导变弱；
2. 回款效率重新恶化；
3. 交付与营运资本压力先于利润恶化。

### 3. 科士达：标准化稳态兑现

升级为“可正式写进接力主线”的条件：

1. 数据中心核心业务收入持续跑赢双主业整体；
2. `AI 订单纯度 / 数据中心 AI 收入` 开始有更清晰拆分；
3. `OCF` 持续改善并保持稳态。

证伪条件：

1. 数据中心增长继续存在，但始终无法证明 AI 纯度；
2. 业绩改善主要回到新能源恢复解释；
3. 预付款、存货和合同负债抬升快于收入兑现。

## 六、组级接力升级线

`AIDC供配电` 只有在下面两组条件同时成立时，才从 `低位观察` 升级为 `补涨加速 / 基建接力`：

### 行情条件

1. 组级或核心样本的 `月线趋势` 转为上升；
2. 组级或核心样本的 `周线趋势` 转为上升；
3. 组级 `20日均值` 转正；
4. 组级 `60日均值` 超过 `30%`；
5. 组内不再只是单一个股偏强，而是至少两条路径出现同步转强。

若只满足日线或 20 日反弹，但月线、周线仍未转为上升趋势，只能写成 `低位修复观察`，不能写成 `基建接力`。

### 经营条件

1. `金盘 / 科华 / 科士达` 里至少一条路径形成 `订单 / 收入 / OCF` 连续闭环；
2. 另外两条路径中至少有一条不拖后腿，证明不是孤立个股行情；
3. AI 订单纯度、数据中心收入拆分、回款与营运资本压力没有明显恶化。

## 九、本版结论

当前最稳妥、也最有研究价值的结论不是“供配电就是下一条主线”，而是：

> `AIDC供配电` 已经具备单独做结果层研究的资格，但它不是单一链条，而是 `订单型高弹性设备`、`平台型智算供电基础设施`、`标准化稳态兑现` 三条路径并行。现在真正要跟踪的是三条路径里谁先把经营验证推进成行情确认，而不是继续笼统讨论“基建会不会补涨”。
