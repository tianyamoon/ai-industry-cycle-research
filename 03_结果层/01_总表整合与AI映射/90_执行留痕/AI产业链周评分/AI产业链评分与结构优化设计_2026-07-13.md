# AI产业链评分与结构优化设计

> 设计日期：2026-07-13
> 目标：在不改变现有五层顶层结构、不大规模搬迁历史文件的前提下，修复 L4 身份冲突，重建当前评分入口，并完成基于最新行情、消息与公告的全量阶段重评分。

## 一、问题定义

当前评分链存在四个会直接影响可靠性和可读性的问题：

1. `337` 条 L4 只有 `336` 个唯一规范主卡；两个不同研究坐标共用“人形机器人”，现有检查器按显示名称去重。
2. 同名 `2026-07-03` 报告分别保留旧闸门 `formal PASS` 和现行闸门 `formal FAIL`，状态口径冲突。
3. 结果层把摘要、337 个 L4 明细、733 行完成清单和数千条机器错误混在一起，日常阅读成本过高。
4. 最新行情已到 `2026-07-13`，但全量评分仍停在 `2026-07-03`；局部补证也没有原子回写唯一状态源。

## 二、目标结构

保留：

`00_总控台 / 01_过程层 / 02_桥接层 / 03_结果层 / 99_交接包`

新增或固定以下职责：

1. `00_总控台/AI产业链L4注册表.csv`
   - 机器权威身份表。
   - 每个 L4 固定一个 `l4_id`；显示名称和 canonical Wiki 页面允许重复。
2. `00_总控台/AI产业链L4样本可信度升级总表.md`
   - 人工可读状态源，增加 `L4_ID`，继续承载 A/B/C、证据层、阻塞与升级动作。
3. `03_结果层/01_总表整合与AI映射/AI产业链当前评分总览.md`
   - 唯一当前阅读入口。
   - 只保留数据截止日、L1/L2 排序、变化、可信度、阻塞和下次触发器。
4. `03_结果层/01_总表整合与AI映射/90_执行留痕/AI产业链周评分/YYYY-MM-DD/`
   - 保存 dated 报告、完成清单、证据增量、周审计和机器检查输出。
5. `tools/ai-chain-scoring/check_weekly_completion.py`
   - 全量覆盖改为按 `l4_id` 校验；拒绝缺失、重复和未知 ID。

## 三、数据契约

L4 注册表最小字段：

```csv
schema_version,l4_id,l1,l2,l3,l4,canonical_topic,research_coordinate,wiki_path
```

完成清单在原字段前增加：

```csv
schema_version,l4_id,l1,l2,l3,l4,canonical_topic,sample_id,parent_l4_id,market,ticker
```

约束：

1. topic 行必须有唯一 `l4_id`。
2. stock 行必须有唯一 `sample_id` 和存在于注册表的 `parent_l4_id`。
3. 样本角色按 `parent_l4_id` 聚合，不再按题材显示名称聚合。
4. `technical_cross_source_ok=是` 只在字段级存在至少两个可比较来源且一致时成立；“调用成功但可比较源不足”必须写 `否`。

## 四、本轮评分方法

本轮定义为 `全量阶段重评分`，不是正式周更。

1. 全部 337 个 L4 逐项进入 manifest，不允许静默遗漏。
2. 产业逻辑与经营证据只接受本轮新增的 S/A 级事实改变分数；B/C 级消息只进入观察或预期层。
3. 行情影响通过明确的 `market_delta` 进入分数变化；没有可靠组级数据的 L4 保持原分，不用全市场下跌机械改写产业逻辑。
4. 每个变化必须拆成 `evidence_delta / valuation_funds_delta / technical_delta`，并保留 `change_reason`。
5. `2026-07-13` 行情跨源结果为“可比较源不足”，技术证据只能进入 stage；不得把 HTTP 成功等同于正式 A。
6. formal 继续按严格门槛运行；失败即如实保留失败原因。

## 五、阅读路径

日常只按以下顺序阅读：

1. `项目状态面板.md`
2. `AI产业链当前评分总览.md`
3. 需要下钻时进入当周目录的阶段报告或完成清单
4. 需要证据缺口时回到 `AI产业链L4样本可信度升级总表.md`

## 六、兼容与迁移

1. 第一阶段不移动结果层其他 45 个活跃文件。
2. 旧 `2026-07-03 formal PASS` 明确标记为“旧闸门历史结果，已失效”，不删除历史事实。
3. 根目录旧大文件不再作为当前入口；由新总览给出有效口径与历史指针。
4. 保留工作区现有未提交修改，不覆盖、不回退。

1. 注册表有且仅有 337 个唯一 `l4_id`。
2. 两个“人形机器人”拥有不同 `l4_id`，可映射到同一 canonical 页面。
3. 新 manifest 无缺失、重复或未知 L4。
4. 全量 stage 机器闸门通过。
5. formal 机器闸门输出真实结果及完整阻塞摘要。
6. 全工具测试通过，且新增身份冲突回归测试。
7. README、状态面板、治理看板和模块索引指向同一当前入口。
8. 公开仓隐私扫描不出现本机绝对路径、私网端点、持仓或私域原文。
