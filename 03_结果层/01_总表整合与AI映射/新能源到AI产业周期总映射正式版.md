## 一句话结论

新能源对 AI 最大的帮助，不是机械预测“AI 会不会再走一遍新能源”，而是提供一套更稳的研究顺序：

`先看事实和第一变量 -> 再看传导和兑现顺序 -> 再审计现金流、ROIC、客户结构和资本配置`

按这套顺序看，AI 当前最接近：

`主升扩散后段 -> 核心资产高位验证 -> 平台承接、资源运营和基础设施协同分层兑现 -> 应用成熟层仍在继续分化`

## 二、新能源四条正式主结论，如何迁移到 AI

### 1. 股价通常领先财务

新能源已经证明：

`股价先交易景气和预期，财务随后确认`

迁移到 AI，当前已被 `05` 多组样本反复验证：

1. 光模块、PCB、核心算力资产往往先被高景气长期化定价。
2. 资源运营、液冷协同、应用成熟层的财务确认往往明显滞后于股价。
3. 不能只看单季利润 headline 来判断阶段。

对应的研究顺序应固定为：

`订单 / ASP / 客户 / 交付 / CapEx -> 毛利率 -> 利润 -> OCF / ROIC`

### 2. 核心资产常在高景气中先见顶

新能源里，核心资产常在景气仍强时先被长期化定价并先进入高位审计。

迁移到 AI：

1. 光模块龙头、核心算力资产、头部平台型基础设施资产最接近这一角色。
2. 这些样本后续更容易先被估值、持仓结构和资金再平衡审计，而不是先被景气证伪。

### 3. 上游资源和材料股常在后半段更强

新能源中，后半段常见的是：

`核心资产先走完 -> 上游资源、材料、协同平台和设备链继续强化`

迁移到 AI：

1. 平台承接层
2. 资源运营层
3. 基础设施协同层

都更接近这种后半段扩散逻辑。

第三批新增两组，正是为了把这层补实：

1. `数据中心运营_算力租赁组`
2. `液冷_散热二线弹性组`

### 4. 真正穿越者必须靠现金流、ROIC、客户结构和资本配置证明

这是新能源对 AI 最重要的最终审计口径。

迁移到 AI 后，最该统一盯住的是：

1. OCF
2. ROIC
3. 客户集中度
4. 订单质量
5. 资本开支与融资结构
6. 资本配置效率

这也是为什么当前 `05` 的很多高位样本，不能直接写成“天然穿越者”。

## 三、AI 图谱坐标已经如何被样本验证

### 1. 核心算力资产层

这层最接近：

`高景气中先被长期化定价的核心资产`

当前已由：

1. 光模块组
2. 核心算力资产样本
3. HBM / 稀缺存储代理样本

支撑。

### 2. 平台承接层

这层最接近：

`主链条确认后承接景气扩散的平台型资产`

当前已由：

1. 先进封装组
2. 国产算力二线_平台对照组
3. 算力设备组

支撑。

### 3. 资源运营层

这是本轮第三批最重要的新补层。

本轮扩样后，这层已经至少分化为四类平台：

1. AIDC 业绩释放样本
2. 稳态 IDC 现金流 + 智算导入样本
3. 云 / IDC 混合平台转型样本
4. 高弹性租赁 / 调度叙事 + 高波动风险样本

这层的关键审计口径固定为：

`上架率 / 利用率 / GPU或算力租赁价格 / 收入口径 / OCF / 折旧 / 融资`

其中：

1. `润泽科技` 代表 AIDC 强释放平台；
2. `数据港` 与 `奥飞数据` 代表稳态现金流与区域扩张的中间层平台；
3. `光环新网` 与 `宝信软件` 代表资源厚度很强、但更早进入利用率和回报率审计的混合/园区平台；
4. `首都在线` 代表高波动智算转型平台，必须把折旧、借款、融资租赁和 OCF 放在结论前面。

截至 `2026-07-02`，这一层已经完成中报前第一轮正式核验：

1. 当前暂未补到 6 个正式样本明确的 `2026H1` 业绩预告；
2. 报表硬口径仍以 `2025年报 + 2026Q1` 为主；
3. 新增正式披露主要强化了 `光环新网`、`奥飞数据` 与 `宝信软件` 的已有判断，但尚不足以推动组内分层重排。

### 4. 基础设施协同层

这层已由：

1. 服务器_温控组
2. 液冷_散热二线弹性组
3. PCB_CCL 组

共同支撑。

这层不能写成天然穿越层，固定审计口径应是：

`订单质量 / 收入确认 / 毛利率 / 回款 / OCF`

截至 `2026-07-02`，这一层也已完成中报前第一轮正式核验：

1. 当前暂未补到两家公司明确的 `2026H1` 业绩预告；
2. 新增正式披露主要强化了 `高澜股份` 的在手订单与毛利率修复；
3. 但在液冷订单拆分、回款和 OCF 继续验证前，仍不宜把二线样本拔高成高确定性平台资产。

### 5. 应用成熟层

这一层已由：

1. 应用成熟_生态验证组
2. 应用继续验证_企业软件组

补出。

当前已明确出现至少两条不同路径：

1. 行业平台型应用路径
2. 金融入口型应用路径

因此不能再把应用层统一写成“还没有验证”。

### 6. 阴影反证层

失败样本_阴影样本组已经说明：

1. 题材外溢不等于经营兑现
2. 重资产扩张不等于高回报
3. 高拥挤往往先放大尾部风险

## 四、AI 当前所处阶段的正式判断

### 1. 不同子链条并不同步

AI 当前不是统一处在单一阶段，而是明显分层：

1. 核心算力资产层更接近 `主升扩散后段 -> 高位验证`
2. 平台承接层更接近 `后半段扩散 -> 早期执行审计`
3. 资源运营层更接近 `AIDC 释放 / 稳态 IDC 转智算导入 / 混合平台转型 / 高波动智算转型` 并存的分化阶段
4. 基础设施协同层更接近 `景气成立 -> 兑现质量分化`
5. 应用成熟层更接近 `真实付费与经营兑现早中期`

### 2. 当前最适合写进上层的总判断

> AI 当前已经从“主题是否真实”转入“谁能把景气转成高质量经营兑现”的阶段。核心资产开始被高位审计，平台承接、资源运营和基础设施协同进入分层兑现，应用成熟层则刚刚从概念导入走向真实付费和真实合同验证。

补充一句更贴近当前维护状态的正式表述：

> 其中，资源运营层与液冷二线层已经不只是“样本补齐”，而是已完成中报前第一轮正式核验；这说明当前 AI 总论主线已进入“方向成立之后，持续审计经营质量而不是继续扩名单”的阶段。

### 3. 资金层当前最稳的口径

当前最稳的写法仍然是：

1. 公募已验证拥挤和再平衡
2. 成交结构已验证高位分化
3. 北向最新日度口径仍待核验

不能写成：

1. 北向已完整闭环
2. 资金已完成全面出清

## 五、当前四条正式主结论

### 1. 股价通常领先财务

AI 当前大部分强样本都已证明：

`市场先交易景气、预期和稀缺性，财务随后确认`

### 2. 核心资产常在高景气中先见顶

当前最接近先被长期化定价、先进入高位审计的，仍是核心算力资产和少数平台资产。

### 3. 上游资源、材料和基础设施协同常在后半段更强

第三批新增两组说明：

1. 资源运营层已经补成方向包
2. 液冷二线协同层已经补出，并进入中报前第一轮正式核验

后半段扩散逻辑已经不能只靠光模块和 PCB 来解释。

### 4. 真正穿越者必须靠现金流、ROIC、客户结构和资本配置证明

这条结论对 AI 的适用性，已经比对“谁更像新能源哪个环节”更重要。

以下缺口不推翻本版阶段性结论，但会限制其成为更强的全链条结论：

6. 当前资源运营层、液冷二线与 AIDC供配电样本的 `2026H1` 明确业绩预告与正式中报实填。

## 七、估值与赔率边界

本次升级后，AI 总论需要明确区分三件事：

1. `产业方向成立`
2. `公司经营兑现`
3. `当前价格仍有赔率`

当前最稳的估值与赔率口径是：

1. 核心算力硬件的产业兑现最清晰，但光模块、PCB/CCL、HBM、先进封装多数已经进入高位估值和高斜率审计，不能再简单写成低位胜率资产。
2. 国产 AI 芯片和应用平台属于高久期资产，产业地位越核心，越要警惕估值已经把多年利润斜率提前写入。
3. AIDC供配电和部分资源运营层因为行情未充分演绎，具备接力观察意义，但前提是订单、收入、回款和 OCF 形成闭环。
4. 液冷二线和半导体设备旁证层更适合放在弹性观察与利润兑现审计里，不能因为题材热度直接上收为高确定性主线。

因此，当前结果层新增固定入口：

`AI全链条估值_预期_拥挤审计总表.md`

后续任何 AI 总论更新，都应先检查这张表里的 `价格隐含假设 / 最先被压缩的预期 / 投资赔率标签`，再决定是否上调结论。

## 八、证伪条件

当前 AI 总论不能只列风险，还必须列“什么情况说明当前判断需要下调”。本版固定三类证伪纪律：

1. 对核心硬件：若收入、利润、毛利率或订单增速无法匹配高位估值，且成交高位放量滞涨，应下调为 `高位审计 / 赔率压缩`。
2. 对资源运营、供配电、液冷：若订单或资源扩张不能转成收入、回款、OCF 和 ROIC，应下调为 `方向成立 / 经营待证伪`。
3. 对应用成熟层：若真实付费、续费、合同负债和 OCF 不能持续改善，应下调为 `场景验证早期`，不能写成成熟穿越。

对应新增固定入口：

`AI关键结论证伪条件总表.md`

后续所有强结论至少要绑定：

`证伪变量 -> 下调触发条件 -> 下一验证时点 -> 需要回写文件`

## 九、宏观与资本成本约束

AI 产业链研究必须纳入资本成本约束，原因是当前主线里同时存在：

1. 高久期估值资产；
2. 高 CapEx 重资产平台；
3. 高拥挤硬件主线；
4. 依赖融资和回款的项目制公司。

当前最稳的宏观口径是：

1. 利率上行或成长风格流动性收缩时，最先受压的是国产 AI 芯片、应用平台、尚未盈利或利润后置的平台型资产。
2. 信用条件收紧时，最先被审计的是资源运营、AIDC / IDC、供配电、液冷、半导体设备等重资产或项目制环节。
3. 风险偏好下降时，高位拥挤的光模块、PCB/CCL、HBM、先进封装可能先压缩赔率，而不是等基本面先转弱。

对应新增固定入口：

`AI宏观与资本成本敏感性框架.md`

后续周更和中报回写时，应至少同步检查 `LPR / SHIBOR / 中债收益率 / PMI / 美国 10 年期国债收益率 / 海外云厂 CapEx` 六类指标。

## 十、本版最终结论

本版已经足以作为当前 AI 总论主入口。
当前最稳的三句话是：

1. AI 当前整体更接近“主升扩散后段 -> 核心资产高位验证 -> 平台承接与协同扩散分层兑现”的并行阶段，而不是全链条统一退潮。
2. 第三批新增的资源运营层和液冷二线层，已经把过去 AI 总论里最缺的两个中间层补上，其中资源运营层已从双卡扩成六样本方向包，并已完成中报前第一轮正式核验。
3. 资源运营层不是单一“算力租赁故事”，而是最接近 `资本开支先行、资本回报率后审计` 的关键风险带；液冷二线则不是天然穿越层，而是订单、毛利率、回款与 OCF 需要一起审计的协同弹性层。
4. 后续真正决定分化的，不只是赛道空间，而是谁能把景气转成 OCF、ROIC、客户结构和资本配置层面的高质量兑现。
